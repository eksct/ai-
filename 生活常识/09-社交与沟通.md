# 社交与沟通技巧

## 有效沟通

### 沟通模型
```
想 → 组织 → 表达 → 接收 → 理解 → 反馈
↔───────── 沟通的循环 ─────────↔
```

### 倾听技巧
1. **专注**：放下手机，眼神接触
2. **不打断**：让对方说完
3. **确认**："你的意思是……对吗？"
4. **共情**："我能理解你的感受"
5. **提问**：开放式问题（"你怎么看？"）

### 表达技巧
- **先说结论**（金字塔原理）
- **使用"我"陈述**而非"你"指责
  - ❌ "你从不考虑我的感受"
  - ✅ "我觉得被忽视了，因为……"
- **具体而非模糊**
  - ❌ "你总是迟到"
  - ✅ "这周你有三次迟到超过 10 分钟"
- **非暴力沟通公式**：
  > 观察到的事实 + 我的感受 + 我的需要 + 我的请求

## 职场沟通

### 向上汇报

**例行汇报结构：**
1. 当前进度（完成了什么）
2. 遇到的问题（是什么 + 已采取的措施）
3. 下一步计划（将要做什么）
4. 需要的支持（需要领导做什么决策/协调）

**汇报技巧：**
- 先说结果，再说过程
- 坏消息不要瞒着，及早汇报
- 带着方案汇报（不是抛出问题）
- 量化成果（用数据说话）

### 跨部门协作
- 明确目标（共同利益）
- 对齐预期（时间、质量、交付物）
- 定期同步（避免信息差）
- 先理后兵（先讲道理，再讲要求）
- 留好邮件、文档记录

### 接受批评
1. **不抬杠**：先听完
2. **确认理解**："我明白了，问题在于……"
3. **改正**："我会……来改进"
4. **感谢**："谢谢你的反馈"

## 冲突处理

### 冲突升级阶梯

```
分歧 → 争论 → 对抗 → 僵局 → 破裂
  ↓       ↓       ↓       ↓      ↓
讨论    听取   降温    暂停    求第三方
```

### 处理步骤
1. **冷静**：情绪上头时暂停对话（"我需要 10 分钟整理一下思路"）
2. **换位**：站在对方角度理解诉求
3. **聚焦问题**：对事不对人
4. **寻找共同点**：先共识再分歧
5. **解决方案**：一起找第三条路

### 不理智时的应对
- 深呼吸（4-7-8 呼吸法：吸 4 秒，屏 7 秒，呼 8 秒）
- 在心里数到 10 再回应
- 离开现场冷静 15 分钟
- 写下来（把情绪写出来，不要发出去）

## 社交网络

### 建立人脉
- 先付出再索取（帮别人忙）
- 保持联系（不只是有事才找）
- 参加行业活动、社群
- 主动自我介绍
- 记住别人的名字和关键信息

### 人脉维护
- 节假日发真实祝福（不是群发）
- 朋友圈互动（点赞、评论）
- 定期约咖啡/吃饭
- 分享有价值的信息
- 祝贺对方的成就

### 界限感
- 不利用人脉做违法/违规的事
- 不要求别人帮超出范围的忙
- 不轻易借钱
- 不把隐私在公开场合讨论

## 谈判技巧

### 基本原则
1. **BATNA**（最佳替代方案）：谈判前明确自己的底线
2. **锚定效应**：先出价的一方占据心理优势
3. **双赢思维**：不追求零和博弈
4. **沉默是金**：谁先开口谁被动

### 薪资谈判
1. 了解市场行情（行业、地区、职级）
2. 不先报数字（让 HR 先出价）
3. 展示价值（不只是"我想要"，而是"我值得"）
4. 谈总包（不只是月薪，包括奖金、期权、福利）
5. 给对方台阶："如果您能给到 XX，我明天就能入职"

## 情商（EQ）提升

### 自我觉察
- 识别自己的情绪（我现在是生气/焦虑/伤心？）
- 知道情绪的来源（为什么会有这种感觉？）
- 管理情绪表达（不压抑，但选择表达方式）

### 同理心
- 不评判别人的感受
- 试着理解对方的处境
- 不急于给建议
- 有时倾听比解决问题更重要

### 社交雷达
- 察觉气氛（注意到别人的表情和肢体语言）
- 知道什么时候该说什么
- 知道什么时候该闭嘴
- 不同场合表现不同（职场/家人/朋友）

## 常见沟通场景

### 道歉
```
公式：对不起 + 承认错误 + 解释原因(简短) + 改正承诺

✅ "对不起，是我搞错了截止时间，导致项目延迟。我已经重新排期，明天下班前补上，以后会设置多个提醒。"
```

### 拒绝
```
公式：感谢 + 原因 + 拒绝 + 替代方案

✅ "感谢你的邀请，不过我这周已经有安排了。下次有机会一定参加。"
✅ "我很想帮忙，但我现在手头有 A 项目的截止很紧。你看要不要找 XX，或者等下周我空下来？"
```

### 请求帮助
```
公式：说明情况 + 具体请求 + 时间

✅ "我在做一个数据分析表格，公式总是报错。你能帮我看看吗？大概需要 10 分钟，现在或者下午都行。"
```

### 给出反馈（SBI 模型）
```
Situation（场景）："在昨天的项目评审会上"
Behavior（行为）："你打断了好几次发言"
Impact（影响）："这让同事觉得想法没有被充分听取"

→ "下次能不能等别人说完再给反馈？"
```

## 沟通风格

| 风格 | 特点 | 适合 | 沟通要点 |
|------|------|------|----------|
| 直接型 | 干脆、不绕弯子 | 上级、平级 | 直奔主题，给选项 |
| 分析型 | 严谨、重视数据 | 技术、财务 | 给数据、给逻辑 |
| 友好型 | 温和、重视关系 | 下属、客户 | 多关心、多鼓励 |
| 表现型 | 热情、重视认可 | 销售、市场 | 给舞台、给赞赏 |

## 沟通避坑

| 场景 | ❌ 不要说 | ✅ 可以说 |
|------|-----------|-----------|
| 被批评时 | "这不是我的错" | "我来看看怎么改进" |
| 别人倾诉 | "你想多了" / "这有什么好难过的" | "我能理解，这确实不容易" |
| 被夸奖 | "没有没有" | "谢谢，多亏了大家的帮助" |
| 请求帮助 | "帮我一下" | "你现在方便吗？想请教一个问题" |
| 工作失误 | "我以为……" | "实际情况是……，我学到了……" |

> **最高级的沟通，不是能言善辩，而是让对方感到被理解和尊重。**
