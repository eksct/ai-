# 包管理与模块

## 概述

Go 1.16+ 默认使用 Modules 管理依赖。每个项目就是一个 module，有自己的 `go.mod` 文件。

## go.mod

```go
module github.com/yourname/image-app

go 1.21

require (
    github.com/wailsapp/wails/v2 v2.8.0
    github.com/nfnt/resize v0.0.0-20180221191011-83c6a9932646
    github.com/rwcarlsen/goexif v0.0.0-20190401172101-9e8deecb76e2
)

require (
    golang.org/x/image v0.0.0-20200119044424-58c23975cae1 // indirect
    golang.org/x/sys v0.0.0-20220114195835-da31bd327af9 // indirect
)
```

### 常用命令

```bash
go mod init                   # 初始化 module
go mod tidy                   # 整理依赖（添加缺失，删除未用）
go mod download               # 下载所有依赖到本地缓存
go mod vendor                 # 创建 vendor 目录（把依赖拷到项目）
go mod verify                 # 校验依赖完整性

go get github.com/xxx/pkg@v1.0.0  # 安装/升级依赖
go get github.com/xxx/pkg@latest  # 最新版
go get github.com/xxx/pkg@none    # 删除依赖
```

## 包的结构

```text
image-app/
├── main.go                    // package main
├── go.mod
├── go.sum                     // 依赖哈希校验
├── internal/                  // 包内私有，外部不可引用
│   ├── processor/
│   │   └── image.go          // package processor
│   └── metadata/
│       └── exif.go           // package metadata
├── pkg/                       // 可导出的公共包
│   └── types/
│       └── image.go          // package types
└── cmd/                       // 多个入口时
    └── cli/
        └── main.go            // package main
```

### 包的命名规范

- 全小写
- 简短（一个词）
- 不要用下划线或驼峰
- 包名和目录名一致

```go
package processor  // 目录 processor/
package metadata   // 目录 metadata/

// 导入时使用
import "image-app/internal/processor"
import "image-app/pkg/types"
```

### main 包

```go
// main 包编译为可执行文件
package main

import "fmt"

func main() {
    fmt.Println("Hello")
}
```

## 内部包（internal）

`internal` 目录是一个特殊的访问控制：

```go
// project-root/
// ├── internal/
// │   ├── auth/
// │   │   └── jwt.go       // package auth
// │   └── db/
// │       └── mysql.go     // package db
// └── main.go

// 同一 module 内可以引用
import "project-root/internal/auth"
import "project-root/internal/db"

// 外部项目无法引用 internal 包（编译报错）
```

**经验：** 所有不希望外部引用的代码放 `internal`。不需要靠注释来约定。

## 包的初始化

```go
package processor

var SupportedFormats []string

// 包级别初始化
func init() {
    SupportedFormats = []string{"jpg", "png", "gif", "webp"}
}

// init 按依赖顺序执行
// 同一个文件中可以有多个 init（按顺序执行）
// 同一个包的不同文件，按文件名顺序执行
```

**不要依赖 init 执行顺序。** 需要显式初始化时，用公开的 Init 函数：

```go
var db *sql.DB

// 手动调用 InitDB，而不是用 init
func InitDB(dsn string) error {
    var err error
    db, err = sql.Open("sqlite", dsn)
    return err
}
```

## 依赖管理原则

### 不要循环依赖

```go
// package a 引用了 b
// package b 引用了 a
// → 编译错误

// 解决：拆出公共的 package c，a 和 b 都引用 c
```

### 最少依赖原则

```go
// ✅ 一个好范例
go list -m all  # 看项目依赖的所有包
```

一个工具类的项目如果引入了 50 个第三方包，说明设计有问题。Go 的标准库已经涵盖了很多功能。

### 版本管理

```bash
# 查看依赖版本
go list -m -versions github.com/gin-gonic/gin

# 升级到次要版本
go get github.com/gin-gonic/gin@v1.8

# 升级到指定 commit
go get github.com/xxx/pkg@commit_sha
```

## go.sum

`go.sum` 文件记录了每个依赖的哈希值，用于**可复现构建**和**安全校验**。

```go
github.com/wailsapp/wails/v2 v2.8.0 h1:abc123...
github.com/wailsapp/wails/v2 v2.8.0/go.mod h1:def456...
```

- 不要手动修改 `go.sum`
- 提交到 git 仓库（确保可复现）
- 如果怀疑 `go.sum` 损坏，删除后 `go mod tidy` 重建

## 代理的配置

```bash
# 国内开发必须配置
go env -w GOPROXY=https://goproxy.cn,https://goproxy.io,direct

# 私有仓库
go env -w GOPRIVATE=github.com/yourcompany/*
# GOPRIVATE 的包不经过代理，直接拉取
```

## Wails 项目的依赖管理

Wails 项目常见的依赖：

```go
require (
    github.com/wailsapp/wails/v2 v2.8.0
    github.com/mattn/go-sqlite3 v1.14.22    // SQLite
    github.com/nfnt/resize v0.0.0-20180221   // 图片缩放
    github.com/rwcarlsen/goexif v0.0.0-2019  // EXIF 读取
    github.com/fsnotify/fsnotify v1.7.0     // 文件监听
)
```

**注意：** Wails 有自己的一套 go.mod 要求。如果 `go mod tidy` 报错，检查 Wails 版本兼容性。
