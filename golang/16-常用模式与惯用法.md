# 常用模式与惯用法

## 设计原理

### Go 没有设计模式

Go 的设计哲学是"简单"——不鼓励你在生产代码里套用 Java 的 23 种设计模式。

```
工厂模式 → 在 Go 里就是 NewXXX() 函数
单例模式 → sync.Once.Do()
建造者模式 → options 模式
策略模式 → 接口 + 函数参数
观察者模式 → channel + select
适配器模式 → func 类型隐式转换
```

## 核心模式

### Options 模式（函数式选项）

Go 最常用的模式之一，适合有大量可选参数的情况：

```go
type Server struct {
    host    string
    port    int
    timeout time.Duration
    logger  *slog.Logger
}

type Option func(*Server)

func WithHost(host string) Option {
    return func(s *Server) { s.host = host }
}

func WithPort(port int) Option {
    return func(s *Server) { s.port = port; }
}

func WithTimeout(t time.Duration) Option {
    return func(s *Server) { s.timeout = t }
}

func NewServer(opts ...Option) *Server {
    s := &Server{
        host:    "localhost",
        port:    8080,
        timeout: 30 * time.Second,
        logger:  slog.Default(),
    }
    for _, opt := range opts {
        opt(s)
    }
    return s
}

// 使用
srv := NewServer(
    WithPort(9090),
    WithTimeout(10*time.Second),
)
```

### 零值有用模式

Go 的零值不是"没有值"，而是"初始值"。利用零值可以让代码更简洁：

```go
// 零值可用的类型
var buf bytes.Buffer         // 空 buffer 可以直接用
var mu sync.Mutex            // 未加锁的 Mutex 可以直接用
var wg sync.WaitGroup        // 零值可以直接 Add/Done/Wait
var logger slog.Logger       // 零值输出到 os.Stderr

// 自定义零值可用类型
type Result struct {
    Code    int
    Message string
    Data    interface{}
}

var zero Result  // zero.Code = 0, zero.Message = "", zero.Data = nil
// 零值可以直接作为"空结果"返回
```

### 不要导出的内容

```go
// Go 的可见性只有"是否大写"——没有 Java 的 public/private/protected

// 小写 = 包内私有
type imageProcessor struct {}  // 包外不能引用

// 大写 = 公开
type ImageProcessor struct {}  // 包外可以引用

// 如果你不想让外部实例化结构体，但想让他们用接口：
type Processor interface {
    Process(path string) (*Image, error)
}

func NewProcessor() Processor {
    return &imageProcessor{}  // 返回接口，外部拿不到具体类型
}
```

### Go 的错误处理惯用写法

```go
// Go 的常见写法是"if err != nil 优先"
// 这不是啰嗦，这是显式

// 文件处理的标准 Go 风格
func CopyFile(src, dst string) error {
    r, err := os.Open(src)
    if err != nil {
        return fmt.Errorf("open %s: %w", src, err)
    }
    defer r.Close()

    w, err := os.Create(dst)
    if err != nil {
        return fmt.Errorf("create %s: %w", dst, err)
    }
    defer w.Close()

    _, err = io.Copy(w, r)
    if err != nil {
        return fmt.Errorf("copy %s -> %s: %w", src, dst, err)
    }

    // Go 不写 else——if 里 return 了后面就是正常路径
    return nil
}
```

```go

// 整个 Go 社区共用一种代码风格
```
