# 结构体与方法

## 结构体（struct）

Go 没有"类"的概念，用 struct + method 实现类似效果。

### 定义

```go
type User struct {
    ID       int64
    Name     string
    Email    string
    Age      int
    // 小写字段是包内私有的
    createdAt time.Time
}
```

### 创建

```go
// 方式一：字面量（按顺序，不推荐）
u := User{1, "Alice", "alice@example.com", 25, time.Now()}

// 方式二：指定字段名（推荐）
u := User{
    ID:    1,
    Name:  "Alice",
    Email: "alice@example.com",
    Age:   25,
}

// 方式三：new
u := new(User)    // 返回 *User，所有字段零值
u.Name = "Alice"

// 方式四：取地址
u := &User{Name: "Alice"}  // 返回 *User
```

### 值接收者 vs 指针接收者

```go
type User struct {
    Name string
    Age  int
}

// 值接收者（不修改原对象）
func (u User) IsAdult() bool {
    return u.Age >= 18
}

// 指针接收者（修改原对象）
func (u *User) Birthday() {
    u.Age++
}

// 使用
u := User{Name: "Alice", Age: 25}
fmt.Println(u.IsAdult())  // true
u.Birthday()              // Age 变为 26
```

**什么时候用指针接收者：**
1. 需要修改接收者
2. 结构体较大（避免复制）
3. 一致性——如果某个方法用指针，最好全部用指针

## 构造函数

Go 没有构造函数，用普通函数代替：

```go
func NewUser(name, email string, age int) *User {
    return &User{
        Name:      name,
        Email:     email,
        Age:       age,
        createdAt: time.Now(),
    }
}
```

协定：`NewXXX` 返回指针，`XXX` 返回值（看场景）。

## 结构体标签（Tag）

```go
type Config struct {
    Host    string `json:"host" yaml:"host" validate:"required"`
    Port    int    `json:"port" yaml:"port" validate:"min=1,max=65535"`
    Debug   bool   `json:"debug" yaml:"debug"`
    DBPath  string `json:"db_path" yaml:"db_path"`
}

// 解析 tag
t := reflect.TypeOf(Config{})
field, _ := t.FieldByName("Host")
fmt.Println(field.Tag.Get("json"))   // "host"
fmt.Println(field.Tag.Get("yaml"))   // "host"
```

**常见 tag 用途：**
- `json` / `yaml` / `xml`：序列化字段名映射
- `validate`：参数校验
- `gorm`：数据库 ORM 字段映射
- `form`：HTTP 表单参数
- `binding`：Gin 框架的绑定

## 结构体的零值和比较

```go
type Person struct {
    Name string
    Age  int
}

// 零值
var p Person
fmt.Println(p.Name)  // ""
fmt.Println(p.Age)   // 0

// 比较（所有字段可比较才可比较）
type Comparable struct {
    A int
    B string
}
fmt.Println(Comparable{1, "a"} == Comparable{1, "a"})  // true

type NotComparable struct {
    S []int  // slice 不可比较
}
// fmt.Println(NotComparable{} == NotComparable{})  // 编译错误
reflect.DeepEqual(a, b)  // 深度比较（任何类型）
```

## 嵌入（Embedding）

Go 用嵌入实现"组合"：

```go
type Animal struct {
    Name string
}

func (a Animal) Speak() string {
    return "..."
}

// Dog 嵌入 Animal，继承其字段和方法
type Dog struct {
    Animal          // 嵌入（不是字段名）
    Breed string
}

// 重写方法
func (d Dog) Speak() string {
    return "汪汪"
}

// 使用
d := Dog{Animal: Animal{Name: "小黑"}, Breed: "金毛"}
fmt.Println(d.Name)          // "小黑"（从 Animal 提升）
fmt.Println(d.Speak())       // "汪汪"
```

**嵌入 vs 子类：** Go 的嵌入是"提升"机制，不是继承。Dog 不能赋值给 Animal，没有多态（接口才有）。

## JSON 序列化

```go
type Image struct {
    ID     int64  `json:"id"`
    Path   string `json:"path"`
    Size   int64  `json:"size"`
    // 忽略空值
    Tag    string `json:"tag,omitempty"`
    // 总是转为字符串
    Width  int    `json:"width,string"`
    // 不导出到 JSON
    Secret string `json:"-"`
}

img := Image{ID: 1, Path: "/tmp/a.jpg", Size: 1024}

// 序列化
data, err := json.Marshal(img)
// {"id":1,"path":"/tmp/a.jpg","size":1024}

// 反序列化
var img2 Image
json.Unmarshal(data, &img2)
```

## 结构体布局与内存优化

```go
// ❌ 不好：字段顺序导致内存对齐填充（8+8+1+7 = 24 字节）
type Bad struct {
    A int64  // 8 字节
    B bool   // 1 字节 + 7 填充
    C int64  // 8 字节
}

// ✅ 好：按类型大小排列（8+8+1 = 17 → 对齐后 24，但更紧凑）
type Good struct {
    A int64  // 8
    C int64  // 8
    B bool   // 1
}
```

大部分场景不需要优化这个，但如果你写高性能组件，字段顺序影响内存布局。
