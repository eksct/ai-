# 并发编程

## 概述

Go 的并发模型基于 **CSP（Communicating Sequential Processes）**——"不要通过共享内存来通信，而应该通过通信来共享内存"。

核心工具：goroutine + channel + sync 包。

## Goroutine

Goroutine 是 Go 的轻量级线程，由 Go runtime 管理，创建成本极低（几 KB 栈空间）。

```go
// 启动一个 goroutine
go func() {
    fmt.Println("在 goroutine 中执行")
}()

// 启动带参数的 goroutine
msg := "hello"
go func(m string) {
    fmt.Println(m)
}(msg)
```

**重要：** goroutine 中的 panic 会导致整个程序崩溃。所以 goroutine 内必须处理 panic：

```go
go func() {
    defer func() {
        if r := recover(); r != nil {
            log.Printf("goroutine panic: %v", r)
        }
    }()
    // 业务逻辑...
}()
```

## Channel

Channel 是 goroutine 之间通信的管道。

### 创建

```go
ch := make(chan int)          // 无缓冲 channel
ch := make(chan int, 10)      // 有缓冲 channel（缓冲区 10 个）
```

### 发送和接收

```go
ch := make(chan int)

// 发送（在 goroutine 中）
go func() {
    ch <- 42    // 向 channel 发送数据
}()

// 接收
value := <-ch  // 从 channel 接收数据
fmt.Println(value)  // 42
```

**无缓冲 channel：** 发送方阻塞直到接收方接收，接收方阻塞直到有数据发送。用于同步。

**有缓冲 channel：** 缓冲区满时发送方阻塞，缓冲区空时接收方阻塞。用于异步。

### 关闭 channel

```go
ch := make(chan int)
close(ch)

// 接收已关闭的 channel
value, ok := <-ch  // ok=false 表示 channel 已关闭且没有数据了

// 遍历 channel（直到关闭）
for v := range ch {
    fmt.Println(v)
}
```

**原则：** 发送方关闭 channel，接收方不要关闭。向已关闭的 channel 发送会 panic。

## Select

`select` 让你同时等待多个 channel 操作：

```go
select {
case msg := <-ch1:
    fmt.Println("收到 ch1:", msg)
case msg := <-ch2:
    fmt.Println("收到 ch2:", msg)
case <-time.After(1 * time.Second):
    fmt.Println("超时了")
default:
    fmt.Println("没有 channel 就绪")
}
```

### 超时控制

```go
select {
case result := <-resultCh:
    fmt.Println(result)
case <-time.After(3 * time.Second):
    fmt.Println("超时了")
}
```

### 退出信号

```go
func worker(stop <-chan struct{}) {
    for {
        select {
        case <-stop:
            fmt.Println("收到退出信号")
            return
        case job := <-jobs:
            process(job)
        }
    }
}

// 通知停止
stop := make(chan struct{})
go worker(stop)

// 停止 worker
close(stop)
```

## 同步原语

### WaitGroup（等待 goroutine 完成）

```go
var wg sync.WaitGroup

for i := 0; i < 5; i++ {
    wg.Add(1)
    go func(i int) {
        defer wg.Done()
        fmt.Println("worker", i)
    }(i)
}

wg.Wait()  // 等待所有 worker 完成
```

**注意：** `wg.Add()` 必须在 goroutine 启动前调用，不要在 goroutine 内部调用。

### Mutex（互斥锁）

```go
type SafeCounter struct {
    mu    sync.Mutex
    count int
}

func (c *SafeCounter) Inc() {
    c.mu.Lock()
    defer c.mu.Unlock()
    c.count++
}

func (c *SafeCounter) Value() int {
    c.mu.Lock()
    defer c.mu.Unlock()
    return c.count
}
```

### RWMutex（读写锁）

```go
type SafeCache struct {
    mu    sync.RWMutex
    data  map[string]interface{}
}

func (c *SafeCache) Get(key string) interface{} {
    c.mu.RLock()          // 读锁，可多个同时读
    defer c.mu.RUnlock()
    return c.data[key]
}

func (c *SafeCache) Set(key string, val interface{}) {
    c.mu.Lock()           // 写锁，互斥
    defer c.mu.Unlock()
    c.data[key] = val
}
```

### Once（只执行一次）

```go
var once sync.Once
var db *sql.DB

func GetDB() *sql.DB {
    once.Do(func() {
        db = openDB()  // 只执行一次，线程安全
    })
    return db
}
```

## 并发模式

### 生产者-消费者

```go
func producer(ch chan<- int) {
    for i := 0; i < 10; i++ {
        ch <- i
    }
    close(ch)
}

func consumer(ch <-chan int, wg *sync.WaitGroup) {
    defer wg.Done()
    for v := range ch {
        fmt.Println("消费:", v)
    }
}

// 使用
ch := make(chan int, 5)
go producer(ch)

var wg sync.WaitGroup
wg.Add(3)
for i := 0; i < 3; i++ {
    go consumer(ch, &wg)
}
wg.Wait()
```

### Fan-out / Fan-in

```go
// fan-out：一个 channel 的数据分发到多个 goroutine 处理
// fan-in：多个 goroutine 的结果合并到一个 channel

func merge(chs ...<-chan int) <-chan int {
    out := make(chan int)
    var wg sync.WaitGroup

    for _, ch := range chs {
        wg.Add(1)
        go func(c <-chan int) {
            defer wg.Done()
            for v := range c {
                out <- v
            }
        }(ch)
    }

    go func() {
        wg.Wait()
        close(out)
    }()

    return out
}
```

### Worker Pool（协程池）

```go
type Pool struct {
    jobs   chan func()
    wg     sync.WaitGroup
}

func NewPool(size int) *Pool {
    p := &Pool{
        jobs: make(chan func(), 100),
    }
    for i := 0; i < size; i++ {
        go func() {
            for job := range p.jobs {
                func() {
                    defer func() {
                        if r := recover(); r != nil {
                            log.Printf("job panic: %v", r)
                        }
                    }()
                    job()
                }()
            }
        }()
    }
    return p
}

func (p *Pool) Submit(job func()) {
    p.wg.Add(1)
    p.jobs <- func() {
        defer p.wg.Done()
        job()
    }
}

func (p *Pool) Shutdown() {
    close(p.jobs)
    p.wg.Wait()
}
```

## 踩坑记录

### 1. goroutine 泄漏

```go
// ❌ goroutine 永远不退出
go func() {
    for {
        select {
        case v := <-ch:
            process(v)
        }
    }
}()

// ✅ 可以正常退出
go func() {
    for {
        select {
        case v := <-ch:
            process(v)
        case <-stop:
            return
        }
    }
}()
```

### 2. 对 map 并发读写

```go
// ❌ fatal error: concurrent map writes
m := make(map[string]int)
go func() { m["a"] = 1 }()
go func() { m["a"] = 2 }()

// ✅ 加锁或用 sync.Map
var m sync.Map
m.Store("a", 1)
m.Load("a")
```

### 3. channel 死锁

```go
// ❌ 无缓冲 channel 在同一个 goroutine 中 send + recv
ch := make(chan int)
ch <- 42    // 阻塞，没人接收
v := <-ch   // 永远不会执行到

// ✅ 分别放在不同 goroutine
```

### 4. 循环变量捕获

```go
// ❌ 所有 goroutine 共享同一个 i
for i := 0; i < 3; i++ {
    go func() {
        fmt.Println(i)  // 可能全是 3
    }()
}

// ✅ 传参
for i := 0; i < 3; i++ {
    go func(i int) {
        fmt.Println(i)
    }(i)
}
```
