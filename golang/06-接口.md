# 接口

## 概述

Go 的接口是**隐式实现**的——不像 Java 那样显式 `implements`。只要类型实现了接口的所有方法，它就自动实现了该接口。

## 接口定义

```go
type Reader interface {
    Read(p []byte) (n int, err error)
}

type Writer interface {
    Write(p []byte) (n int, err error)
}

// 组合接口
type ReadWriter interface {
    Reader
    Writer
}
```

## 隐式实现

```go
type FileReader struct {
    path string
}

// 实现了 Reader 接口（不需要写 implements）
func (r *FileReader) Read(p []byte) (n int, err error) {
    // ...
    return
}

// 使用：任何实现了 Reader 的类型都可以传进来
func processData(r Reader) {
    buf := make([]byte, 1024)
    r.Read(buf)
}
```

**好处：** 不用提前规划接口，先写具体类型，后面自然发现接口。这是 Go 的设计哲学——"被需要的接口才是好接口"。

## 接口值

接口在 Go 内部是一个二元组（type, value）：

```go
var r Reader = &FileReader{}
// r 内部存储：(type: *FileReader, value: 指针)
```

```go
var r Reader = nil        // (nil, nil)
var p *FileReader = nil
var r Reader = p          // (type: *FileReader, value: nil)
// 注意：r != nil！因为 type 不为空
```

这个坑经常出现：
```go
func getReader() Reader {
    var p *FileReader = nil
    return p  // 返回的不是 nil
}

r := getReader()
fmt.Println(r == nil)  // false！
```

**修复：** 显式返回 nil
```go
func getReader() Reader {
    if failed {
        return nil  // 直接返回 nil 接口值
    }
    return &FileReader{}
}
```

## 类型断言

```go
var r Reader = &FileReader{}

// 安全断言
f, ok := r.(*FileReader)
if ok {
    fmt.Println(f.path)
}

// 类型 switch
switch v := r.(type) {
case *FileReader:
    fmt.Println("FileReader:", v.path)
case *BufReader:
    fmt.Println("BufReader:", v.size)
default:
    fmt.Println("未知类型")
}
```

## 空接口

```go
// 空接口可以表示任何类型
var any interface{}
any = 42
any = "hello"
any = &User{}

// Go 1.18+ 可以用 any 代替 interface{}
var any any

// 常用场景：JSON 解析
data := `{"name": "Alice", "age": 25}`
var result map[string]interface{}
json.Unmarshal([]byte(data), &result)
```

## 接口设计原则

### 小接口优先

```go
// Go 标准库的接口都很小
type error interface {
    Error() string
}

type Stringer interface {
    String() string
}

type Reader interface {
    Read(p []byte) (n int, err error)
}
```

**经验：** 一个接口有 1-3 个方法就够了。超过 3 个方法，考虑拆分成多个小接口。

### 接受接口，返回结构体

```go
// ✅ 常见的 Go 模式
func process(r io.Reader) ([]byte, error) {  // 接受接口
    data, _ := io.ReadAll(r)
    return data, nil
}

// ❌ 不返回接口（除非是 error、io.Reader 等）
func NewReader(r io.Reader) io.Reader {
    return &bufio.Reader{r: r}
}
```

### 接口放在使用方，不是实现方

```go
// 定义接口的地方
type UserStore interface {
    Get(id int) (*User, error)
    List() ([]*User, error)
}

// 使用方（service）定义接口
// 实现方（database）不需要知道这个接口

type UserService struct {
    store UserStore  // 依赖接口
}
```

## 在 Wails 中接口的应用

```go
// 定义图片处理器接口
type ImageProcessor interface {
    Resize(path string, width, height int) error
    GetThumbnail(path string) ([]byte, error)
    GetMetadata(path string) (*ImageMeta, error)
}

// 不同的实现
type GoImageProcessor struct {
    // 标准库实现
}

type CgoImageProcessor struct {
    // 用 C 库实现（性能更好）
}

// Wails 绑定
type App struct {
    processor ImageProcessor  // 依赖接口，可替换
}
```

## 接口性能

```go
// 接口调用有间接成本（动态分发）
// 极端性能场景（每秒百万次调用）可以考虑避免接口
// 一般业务代码完全不用在意
```
