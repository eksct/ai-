# 函数

## 函数声明

```go
func add(a int, b int) int {
    return a + b
}

// 类型相同时可以简写
func add(a, b int) int {
    return a + b
}
```

## 多返回值

Go 函数可以返回多个值——这是它和 C/Java 最大的区别之一。

```go
// 最常见的模式：返回结果 + error
func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, errors.New("除数不能为 0")
    }
    return a / b, nil
}

// 调用
result, err := divide(10, 0)
if err != nil {
    log.Fatal(err)
}

// 命名返回值
func findUser(id int) (user User, err error) {
    // user 和 err 已经在返回值中声明了
    user, err = db.Find(id)
    return  // 裸返回，不需要写 user, err
}
```

**经验：** 命名返回值在小函数里可读性好，大函数里不建议——你不知道返回值在哪被更新的，不如显式写 `return`。

## 可变参数

```go
func sum(nums ...int) int {
    total := 0
    for _, n := range nums {
        total += n
    }
    return total
}

sum(1, 2, 3)       // 6
sum([]int{1,2,3}...) // 展开切片
```

## 函数作为值

```go
// 函数是一等公民，可以赋值给变量
fn := func(a, b int) int { return a + b }
result := fn(1, 2)

// 作为参数（回调）
func mapSlice(arr []int, fn func(int) int) []int {
    result := make([]int, len(arr))
    for i, v := range arr {
        result[i] = fn(v)
    }
    return result
}

double := func(x int) int { return x * 2 }
mapped := mapSlice([]int{1,2,3}, double)  // [2,4,6]
```

## 闭包

```go
// 闭包：捕获外部变量的函数
func counter() func() int {
    count := 0
    return func() int {
        count++
        return count
    }
}

c := counter()
fmt.Println(c())  // 1
fmt.Println(c())  // 2
fmt.Println(c())  // 3

// 陷阱：闭包捕获的是变量引用，不是值
funcs := make([]func(), 0)
for i := 0; i < 3; i++ {
    funcs = append(funcs, func() {
        fmt.Println(i)  // 全部打印 3
    })
}
// 修复：每次循环创建新变量
for i := 0; i < 3; i++ {
    i := i  // 重新声明
    funcs = append(funcs, func() {
        fmt.Println(i)  // 0, 1, 2
    })
}
```

## init 函数

```go
// 每个包可以有多个 init，在 main 之前自动执行
// 按依赖顺序执行

var db *sql.DB

func init() {
    var err error
    db, err = sql.Open("sqlite", "data.db")
    if err != nil {
        log.Fatal(err)
    }
}
```

**注意：** 不要依赖 init 的执行顺序。如果需要严格控制初始化顺序，用显式调用代替 init。

## 恐慌（panic）和恢复（recover）

```go
// panic：程序崩溃（类似 Java 的 RuntimeException）
func must(err error) {
    if err != nil {
        panic(err)
    }
}

// recover：捕获 panic（类似 try-catch，但只能用在 defer 中）
func safeDoSomething() {
    defer func() {
        if r := recover(); r != nil {
            log.Printf("recovered from panic: %v", r)
        }
    }()
    doSomething()  // 如果这里 panic，会被上述 recover 捕获
}
```

**生产原则：** 不要用 panic/recover 代替错误处理。只有"程序真的无法继续"的情况才 panic（比如端口被占用、数据库连不上）。业务逻辑错误用 `error` 返回值。

## 函数类型

```go
// 定义函数类型
type Handler func(w http.ResponseWriter, r *http.Request)

// 函数类型可以实现接口
type HandlerFunc func(http.ResponseWriter, *http.Request)

func (f HandlerFunc) ServeHTTP(w http.ResponseWriter, r *http.Request) {
    f(w, r)
}
```

## 延迟调用（defer）执行顺序

```go
func example() {
    defer fmt.Println(1)
    defer fmt.Println(2)
    defer fmt.Println(3)
    // 输出：3, 2, 1（栈顺序）
}
```

defer + 闭包常见陷阱：
```go
func main() {
    for i := 0; i < 3; i++ {
        defer func() {
            fmt.Println(i)  // 全部输出 3
        }()
    }
}

// 修复
func main() {
    for i := 0; i < 3; i++ {
        i := i  // 重新声明
        defer func() {
            fmt.Println(i)  // 2, 1, 0
        }()
    }
}
```
