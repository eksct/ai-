# 数据库工具

## 概述

GoLand 内置了 DataGrip 的核心功能——不需要单独开一个数据库客户端。你的图像管理 App 如果用 SQLite 存标签/分类信息，直接在 IDE 里操作就行。

## 连接数据库

### 添加数据源

```
View → Tool Windows → Database
右侧面板 → + → Data Source → SQLite
```

SQLite 配置：
```
File: 选择项目中的 .db 文件
Driver: SQLite（GoLand 会自动下载）
```

也支持：
```
MySQL / PostgreSQL / SQL Server / MongoDB / Redis
+ → Data Source → 选对应的数据库
```

### 测试连接

填完连接信息后点 **Test Connection**，不通的话看右下角的报错原因——通常是因为数据库没启动、端口不对、驱动没下载。

## 常用操作

### 查询

```
双击表名 → 显示表内容（自动执行 SELECT *）
右键表 → SQL Scripts → Select All（生成 SELECT 到编辑器）
在 .sql 文件中 Ctrl+Enter → 执行 SQL
```

### 编辑数据

```
直接在查询结果表格中双击单元格修改
Ctrl+Enter → 提交修改
Ctrl+\ → 放弃修改
```

### 查看表结构

```
右键表 → Modify Table...（或选中表按 F4）
├── Columns（字段名、类型、默认值、是否 NULL）
├── Indexes（索引信息）
├── Foreign Keys（外键约束）
└── Triggers（触发器）
```

### 导出数据

```
右键查询结果 → Export Data
├── CSV（最常用）
├── JSON（给前端或 API 用）
├── SQL INSERT（备份或迁移）
└── Markdown（写文档用）
```

## SQL 文件管理

GoLand 会把 SQL 文件当代码一样管理——语法高亮、自动补全、格式化。

### 保存常用查询

在项目中创建 `.sql` 文件：

```sql
-- queries/images.sql
-- :name get_images_by_tag
SELECT i.* FROM images i
JOIN image_tags it ON i.id = it.image_id
JOIN tags t ON it.tag_id = t.id
WHERE t.name = $1  -- PostgreSQL 参数格式
ORDER BY i.created_at DESC;

-- :name count_images
SELECT COUNT(*) FROM images;
```

### SQL 格式化

```
Ctrl+Alt+L 格式化 SQL
```

比手动对齐 SQL 快很多。

## 与 Go 代码联动

### 从 SQL 生成 Go struct

```sql
-- 写 SQL
CREATE TABLE images (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    path TEXT NOT NULL,
    size INTEGER,
    width INTEGER,
    height INTEGER,
    format TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

```
右键编辑区 → Tools → Go → Generate Struct from SQL
生成：
```

```go
type Images struct {
    ID        int       `json:"id" gorm:"column:id;primaryKey;autoIncrement"`
    Name      string    `json:"name" gorm:"column:name;not null"`
    Path      string    `json:"path" gorm:"column:path;not null"`
    Size      int       `json:"size" gorm:"column:size"`
    Width     int       `json:"width" gorm:"column:width"`
    Height    int       `json:"height" gorm:"column:height"`
    Format    string    `json:"format" gorm:"column:format"`
    CreatedAt time.Time `json:"created_at" gorm:"column:created_at;default:CURRENT_TIMESTAMP"`
}
```

但这个生成比较基础。实际项目中我建议用 `sqlc` 或手写，比自动生成的可控性高。

### 从 Go struct 生成 CREATE TABLE

反过来也支持：在 Go struct 上 `Alt+Insert` → Generate SQL DDL。

## 查询控制台（Query Console）

```
右键数据源 → Open Query Console
```

多个查询可以同时运行：
```sql
-- 选中一段执行
SELECT * FROM images LIMIT 10;
-- 选中另一段执行
SELECT COUNT(*) FROM tags;
```

每个查询的结果在不同的标签页显示。

## 踩坑记录

### 1. SQLite 文件被锁定

```
[SQLITE_BUSY] The database file is locked
```

**原因：** 你的 Go 应用正在打开数据库，GoLand 也在操作同一个文件

**解决：**
- 在 GoLand 中做查询时确保应用没有占用数据库
- 如果用 WAL 模式可以缓解这个问题：
```sql
PRAGMA journal_mode=WAL;
```

### 2. 数据库路径搞错

Wails 打包后路径变了，你连的 `./data.db` 可能是构建目录下的，不是项目目录下的。

**解决：** 在 Go 代码里把数据库路径打印到日志，然后在 GoLand 中用这个完整路径连接。

### 3. GoLand 连不上 MySQL

常见原因：
- MySQL 没启动 → `brew services start mysql`（Mac）/ `net start mysql`（Windows）
- 驱动没下载 → 点数据源配置的下载链接手动下
- Host 写成了 `localhost` → 有时候要改成 `127.0.0.1`（MySQL 的 socket 问题）
