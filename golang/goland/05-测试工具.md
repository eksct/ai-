# 测试工具

## 概述

GoLand 的测试集成是它比 VS Code 好的另一个点。不用切终端、不用敲命令，测什么、覆盖率多少一目了然。

## 运行测试

### 基本操作

```go
// 文件名必须是 _test.go 结尾
// image_processor_test.go

func TestResizeImage(t *testing.T) {
    result := ResizeImage("test.jpg", 800)
    assert.NotNil(t, result)
    assert.Equal(t, 800, result.Bounds().Max.X)
}
```

```
运行方式：
  行号左侧绿色三角 → Run / Debug
  Ctrl+Shift+F10      运行当前上下文中的测试
  右键测试文件名      运行文件中所有测试
  右键目录            运行包下所有测试
```

### 运行单个测试

```go
// 方法一：光标放在函数名上 → Ctrl+Shift+F10
// 方法二：Options → Run Context Configuration → 选择测试
```

### 重复运行

```
右键测试 → Run All Checked Tests
适合：fix 了一个 bug 后跑全部相关测试
```

## 测试结果面板

```
Test Results
├── Passed: 12
├── Failed: 1
│   ├── TestResizeLargeImage
│   │   ├── Error: assertion failed
│   │   └── Stack trace (点击跳转到失败行)
└── Skipped: 0
```

**实用功能：**
- 失败的测试会高亮，双击跳转到失败行
- `Rerun` 只跑失败测试（不用全跑）
- `Export Test Results` 导出测试报告

## 覆盖率

### 运行带覆盖率的测试

```
Run → Run with Coverage
或右键测试目录 → More Run/Debug → Run with Coverage
```

### 覆盖率视图

```
Coverage 面板
├── Classes: 65% (15/23 functions covered)
├── Lines: 72%
└── 绿色：已覆盖 / 红色：未覆盖
```

点击百分比可以逐行看哪些代码没测到——比手动翻代码找覆盖盲区快多了。

### 覆盖率门槛

```bash
# 也可以用 CLI 检查覆盖率
go test -coverprofile=coverage.out ./...
go tool cover -html=coverage.out -o coverage.html
```

但在 GoLand 里直接 Run with Coverage 看红绿标注更直观。

## 生成测试模板

```go
// 在函数上按 Alt+Insert → Test for function
// GoLand 会生成：

func TestNewImageProcessor(t *testing.T) {
    type fields struct {
        cache map[string]*Image
    }
    type args struct {
        cacheDir string
    }
    tests := []struct {
        name    string
        fields  fields
        args    args
        want    *ImageProcessor
        wantErr bool
    }{
        // TODO: Add test cases.
    }
    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            // ...
        })
    }
}
```

Table-driven test 是 Go 的标准测试模式，GoLand 原生支持。

## 断点调试测试

和调试普通代码一样——设断点后点测试行号旁边的 Debug 按钮。

```go
func TestProcessImage(t *testing.T) {
    result, err := ProcessImage("test.jpg")  // ← 设断点
    // 按 F7 步入 ProcessImage
    // 按 F8 步过，一行一行看
    // 按 Alt+F8 看表达式值
}
```

## 性能测试（Benchmark）

```go
func BenchmarkResizeImage(b *testing.B) {
    for i := 0; i < b.N; i++ {
        ResizeImage("test.jpg", 800)
    }
}
```

```
运行方式：和普通测试一样
结果面板会显示：
BenchmarkResizeImage-8   1000   1234567 ns/op
                                        ↑ 每次操作耗时
```

## Wails 项目的测试注意事项

Wails 项目中测试 Bind 方法时，需要注意 context 的问题：

```go
// ❌ 直接测会 panic（ctx 是 nil）
func TestProcessImage(t *testing.T) {
    app := &App{}
    app.ProcessImage("test.jpg")  // 调了 runtime.LogInfo(a.ctx...) → nil panic
}

// ✅ 注入一个 mock context
import "context"

func TestProcessImage(t *testing.T) {
    app := &App{}
    app.ctx = context.Background()  // 或者用 context.TODO()
    // 但注意：context.Background() 是空的，调 runtime 方法会报错
    // 最好把业务逻辑抽出来单独测试
}
```

**推荐做法：** 业务逻辑和数据操作与 Wails Runtime 分离，Bind 方法只做编排。

```go
// service 层不依赖 context，可以单独测试
type ImageProcessor struct {}

func (p *ImageProcessor) Resize(path string, size int) (*Image, error) {
    // 纯业务逻辑
}

// Bind 层只做桥接
func (a *App) ResizeImage(path string, size int) (*Image, error) {
    runtime.LogInfo(a.ctx, "调整图片: "+path)
    return a.processor.Resize(path, size)
}
```

## 测试命令行

GoLand 的终端里也可以用命令行，不过 IDE 集成更方便：

```bash
# 跑全部测试
go test ./...

# 跑指定包
go test ./internal/processor/...

# 只看失败
go test ./... | grep -E "FAIL|PASS|ok"

# 带覆盖率的详细报告
go test -v -cover ./... | tee test_report.txt
```
