# 插件与生产力工具

## 概述

GoLand 本身已经很强大，但加上几个关键插件，效率还能再上一个台阶。不过别装太多——装多了启动变慢，反而降低效率。

## 推荐的插件

### 必须装

| 插件 | 作用 |
|------|------|
| **Key Promoter X** | 你用了鼠标操作后，它会提示对应的快捷键是什么——帮你养成快捷键习惯 |
| **Rainbow Brackets** | 不同层级的括号用不同颜色显示，嵌套代码一眼看清 |
| **GitToolBox** | 在编辑器中显示当前行是谁写的、git blame 信息 |

### 推荐装

| 插件 | 作用 |
|------|------|
| **Goanno** | 自动生成 Go 注释模板，输入 `//` 按 Tab 补全 |
| **.env files** | 环境变量文件语法高亮 |
| **String Manipulation** | 字符串格式转换（驼峰、下划线、大小写互转） |
| **CodeGlance Pro** | 右侧显示代码缩略图（类似 VS Code 的 minimap） |
| **WakaTime** | 统计你写代码的时间分布，看自己每天写了多久 |

### 不用装

| 插件 | 原因 |
|------|------|
| **Lombok** | Go 不需要 |
| **Alibaba Java Coding Guidelines** | Go 项目用不上 |
| **SonarLint** | GoLand 自带的代码检查已经够用 |
| 各种 Theme 插件 | GoLand 自带的 Darcula 就是最好的主题 |

## 插件安装

```
File → Settings → Plugins → Marketplace → 搜索 → Install
```

也可以从 JetBrains 插件市场下载后本地安装：
```
Settings → Plugins → ⚙ → Install Plugin from Disk...
```

## 内建工具

GoLand 自带了一些你可能不知道的功能：

### HTTP Client（代替 Postman）

在项目中创建 `.http` 或 `.rest` 文件：

```http
### 打开图片
GET http://localhost:8080/api/images/123
Accept: application/json

### 创建标签
POST http://localhost:8080/api/tags
Content-Type: application/json

{
  "name": "风景",
  "color": "#00ff00"
}

### 批量导入
POST http://localhost:8080/api/images/batch
Content-Type: application/json

[
  "/Users/me/Photos/photo1.jpg",
  "/Users/me/Photos/photo2.jpg"
]
```

行号旁边有绿色箭头，点击直接发请求，返回结果在下方面板显示。不需要切到 Postman。

### 终端

```
Alt+F12 打开终端
```

支持多 session：
- 分屏：右键标签 → Split Right
- 新 session：+ 号按钮

### Scratch File（临时文件）

```
Ctrl+Shift+N → 选择 Go 或 SQL → 写临时代码
```

不会保存到项目目录，适合：
- 测试一段 Go 代码逻辑
- 写一个临时 SQL 查询
- 记 JSON 格式化结果

### 本地历史

```
右键文件 → Local History → Show History
```

即使没有 commit，GoLand 也会自动保存文件的历史版本。适合手抖改了代码想找回旧版本。

### 结构搜索和替换

```
Ctrl+Shift+S（结构化搜索）
Ctrl+Shift+M（结构化替换）
```

高级用法：搜索所有 `if err != nil { return err }` 的地方，统计异常处理覆盖率。不过一般用不上。

## Wails 开发实用设置

### 前端预览

Wails 的前端框架（Vue/React）通常需要 dev server，可以在 GoLand 的 Terminal 里跑：

```
# 终端 1：wails dev
# 终端 2：前端 dev server（如果 Wails 没自动启动）
```

两个终端分屏显示，一个看后端日志，一个敲前端命令。

### 文件观察器

如果项目有自动生成的代码（比如 Wails 的 wailsjs），GoLand 可能 index 后报红。

```
右键 wailsjs/ 目录 → Mark Directory as → Excluded
```

这样 GoLand 不会 index 这个目录，IDE 响应更快。

### 前端代码支持

GoLand 对前端框架的支持不如 WebStorm 或 VS Code。如果你做大量前端开发：

- 方案一：给前端代码装 WebStorm 的对应插件（Settings → Plugins → 搜 Vue/React/TypeScript）
- 方案二：前端改动用 VS Code，Go 后端用 GoLand

大多数 Wails 项目的后端逻辑比前端重，GoLand 做主力，前端用 VS Code 补位也是常见的工作流。

## 性能优化

GoLand 吃内存，大项目久了会卡：

```text
Help → Change Memory Settings
推荐：2048 MB（2GB）
如果项目大：4096 MB（4GB）
```

还卡的话：

```text
File → Invalidate Caches and Restart...
清除缓存重启，通常能解决大部分卡顿
```
