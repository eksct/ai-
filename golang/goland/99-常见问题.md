# GoLand 常见问题

## 1. IDE 启动慢

**原因：** 缓存太多、插件太多、索引范围太大

**解决：**
```
File → Invalidate Caches and Restart → Invalidate and Restart
（建议每 1-2 个月做一次）

Settings → Appearance & Behavior → System Settings → Project Opening
  → Reopen projects on startup: 关掉（不恢复上次的项目，启动快很多）
  
标记不用的目录为 Excluded（见《Wails 项目配置》篇）
```

## 2. 代码补全卡顿 / 索引卡住

**原因：** GoLand 在 index 大目录

**解决：**
- 确认 `frontend/node_modules/` 标记为 Excluded
- 确认 `vendor/` 如果不是必须就加上 `.gitignore`
- `File → Invalidate Caches` 清掉索引重建

## 3. 中文输入法问题

**现象：** 输入中文时候选框不跟随光标、候选框在屏幕左上角

**解决：**
- 升级 JetBrains Runtime 到最新版（`Help → Check for Updates`）
- 换用自带的中文输入法（Windows 微软拼音 / Mac 自带中文输入法）
- 如果还不行，在 `Help → Edit Custom VM Options` 中添加：
```
-Drecreate.x11.input.method=true
```
（Mac/Linux 适用）

## 4. 快捷键失灵

**原因：** 和系统或其他软件的快捷键冲突

**排查：**
```
Settings → Keymap → 搜索 → 看哪个快捷键被占用了
常见的冲突：Ctrl+Space（系统输入法切换）
```

**解决：**
```
百度输入法：设置 → 按键 → 取消 Ctrl+Space 切换
搜狗输入法：属性设置 → 按键 → 简繁切换取消 Ctrl+Shift+F
```

或者改 GoLand 的快捷键：
```
Settings → Keymap → 搜索 "Basic" → 改为 Alt+/
```

## 5. Wails 项目显示找不到 go.mod

**现象：** GoLand 打开 Wails 项目后，所有 Go 文件都标红

**原因：** GoLand 默认打开目录找 `go.mod`，如果在子目录开项目找不到

**解决：**
- `File → Open` 打开包含 `go.mod` 的根目录
- 如果已经开了，`File → Settings → Go → Go Modules → Enable Go modules integration`

## 6. 终端乱码

**现象：** 中文显示为 `???` 或乱码

**解决：**
```
Settings → Tools → Terminal → Default encoding: UTF-8
（Mac/Linux 一般不需要改，Windows 如果乱码改）
```

## 7. GoLand 和 VS Code 的选择

```
场景                               推荐
──────────────────────────────────────────
纯 Go 后端开发                     GoLand
Wails 后端为主，少量前端            GoLand（前端偶尔切 VS Code）
Wails 前后端对半                  VS Code
新学 Go 语言，预算有限             VS Code
需要数据库/HTTP Client 内置         GoLand
项目纯前端（Vue/React）            VS Code/WebStorm
```

**一句话：** GoLand 在 Go 后端开发和测试体验上完胜，但前端支持不如 VS Code。
