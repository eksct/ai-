# 重构与代码分析

## 概述

重构是 GoLand 相比 VS Code 最核心的优势。手改变量名找引用是初级开发者的做法，高级开发者用 IDE 重构。

## 重构操作

### 重命名（Shift+F6）

```go
// 光标放在变量上 → Shift+F6 → 输入新名称 → Enter
// GoLand 会：
// 1. 修改所有引用位置
// 2. 自动处理 Go 的短变量声明（:=）
// 3. 跳过外部依赖中的引用

// 重命名前
func processImage(p string) string {  // p 不直观
    data, _ := os.ReadFile(p)
    return string(data)
}

// 重命名后（p → path）
func processImage(path string) string {  // 清晰多了
    data, _ := os.ReadFile(path)
    return string(data)
}
```

**注意：** Go 的 `:=` 声明中重命名会把 `=` 左边的变量名一起改，但不会影响 `=` 右边的表达式。

### 提取变量（Ctrl+Alt+V）

```go
// 选中表达式 → Ctrl+Alt+V → 自动提取为变量

// 重构前
result = Image{Name: file.Name(), Path: filepath.Join(baseDir, file.Name())}

// 重构后
fileName := file.Name()
result = Image{Name: fileName, Path: filepath.Join(baseDir, fileName)}
```

### 提取方法（Ctrl+Alt+M）

```go
// 选中一段代码 → Ctrl+Alt+M → 输入方法名

// 重构前
func (a *App) ProcessImages(images []string) {
    for _, img := range images {
        file, _ := os.Open(img)
        cfg, _, _ := image.DecodeConfig(file)
        if cfg.Width > 4000 || cfg.Height > 4000 {
            a.resizeImage(img)
        }
        file.Close()
    }
}

// 重构后
func (a *App) ProcessImages(images []string) {
    for _, img := range images {
        if a.isLargeImage(img) {  // 语义清晰
            a.resizeImage(img)
        }
    }
}

func (a *App) isLargeImage(path string) bool {
    file, _ := os.Open(path)
    defer file.Close()
    cfg, _, _ := image.DecodeConfig(file)
    return cfg.Width > 4000 || cfg.Height > 4000
}
```

### 内联（Ctrl+Alt+N）

```go
// 与提取相反，把变量/函数直接替换为表达式
// 光标放在变量上 → Ctrl+Alt+N

name := getUserName()
result := "Hello, " + name
// 内联后
result := "Hello, " + getUserName()
```

### 签名变更（Ctrl+F6）

```go
// 光标放在函数名上 → Ctrl+F6 → 调整参数

// 重构前
func (a *App) SaveImage(path string) error

// 改为
func (a *App) SaveImage(path string, quality int) error
// 所有调用处自动添加 quality 参数（初始为 0，需要你手动改）
```

### 移动和复制（F6 / F5）

```go
// 把 struct 或函数移动到另一个文件/包
// 光标在类型名上 → F6 → 选择目标文件
// GoLand 会自动：
// - 移动代码
// - 更新 import
// - 不破坏调用方
```

## 代码检查（Inspections）

GoLand 在写代码时会实时检查。重点关注：

```go
// ⚠️ 红线：编译错误
// ⚠️ 黄线：警告
// ⚠️ 绿色波浪：弱警告或风格问题
```

### 手工触发检查

```
Code → Inspect Code
Code → Run Inspection by Name → Ctrl+Alt+Shift+I
```

### 常见的 Go 检查项

| 检查项 | 说明 |
|--------|------|
| `Unused parameter` | 函数参数没用到 |
| `Unused variable` | 变量声明了没用 |
| `Unreachable code` | 永远不会执行的代码 |
| `Suspicious assignment` | 可能写错的赋值（如 `=` 写成 `==`） |
| `Error return not checked` | 调用了返回 error 的函数但忽略了 |
| `Redundant type declaration` | 类型声明多余 |
| `Inefficient type assertion` | 类型断言可以优化 |

## Structure 面板

`Ctrl+F12` 打开文件结构，比滚动看代码快得多：

```
App
├── Fields
│   ├── ctx context.Context
│   ├── db *sql.DB
│   └── cache sync.Map
├── Methods
│   ├── startup(ctx context.Context)
│   ├── shutdown(ctx context.Context)
│   ├── ProcessImages(images []string)
│   ├── SaveImage(path string) error
│   └── ...
└── Unexported
    ├── isLargeImage(path string) bool
    └── resizeImage(path string)
```

可以直接在这里搜索函数名、拖动调整顺序、右键重构。

## TODO 管理

GoLand 可以识别 Go 注释中的 TODO：

```go
// TODO: 支持 RAW 格式解码
// FIXME: 这个函数在大文件下 OOM
// HACK: 临时绕过 Windows 上的 WebView2 限制
```

```
View → Tool Windows → TODO
显示所有 TODO，按文件分组，支持过滤
```
