# 调试技巧

## 概述

GoLand 的调试器比打印日志 debug 的效率高一个数量级。尤其是 Wails 项目，Go 后端逻辑复杂时，断点调试比猜日志快得多。

## 基本断点

### 行断点

点击行号右侧空白处（或 `Ctrl+F8`）设置断点：

```go
func (a *App) ProcessImage(path string) *ImageResult {
    file, err := os.Open(path)  // ← 设断点
    if err != nil {
        return nil, err  // ← 想看 err 的内容也设一个
    }
    defer file.Close()
    // ...
}
```

### 条件断点

右键断点，输入条件表达式：

```
条件：path.Contains("large")
仅在文件名包含 "large" 时停住
```

### 表达式求值

停在断点时可以 `Alt+F8` 输入表达式：
```go
len(images)              // 查看切片长度
filepath.Base(path)     // 计算文件名
runtime.GOOS            // 查看系统
```

## Wails 项目的调试

Wails 项目的 debug 比较特殊——你不能直接 Run Go Build 然后 Debug，因为 WebView 需要 Wails CLI 启动。

### 方式一：Attach 到进程

1. 终端先跑 `wails dev`
2. GoLand 中 `Run → Attach to Process`（或 `Ctrl+Alt+F5`）
3. 选择 `wails` 或你的应用进程
4. 断点生效

优点：不需要额外配置
缺点：启动阶段的代码（OnStartup）断点会错过

### 方式二：Go Remote

```
Run → Edit Configurations → + → Go Remote
Host: localhost
Port: 2345（默认）
```

然后终端用 debug 模式启动：
```bash
wails dev -debugger
```

之后点击 Debug 按钮连接，这种方式可以在启动阶段就捕捉断点。

## 调试 Wails Bind 调用

当你在前端调一个 Go 方法，结果不对但不知道哪一步错了：

1. 在 Bind 方法的第一行设断点
2. 前端页面触发调用
3. 断点停住后，按 `F7` 步入，一行一行看数据变化
4. 在 Variables 面板看参数值

**常见问题：**
- 断点没停 → Bind 的包名不对（检查 main.go 里的 Bind 列表）
- 参数是空 struct → 前端传的 JSON 字段名和 Go struct tag 不匹配

## 多 goroutine 调试

GoLand 的调试器会显示当前 goroutine 列表：

```
Debugger 面板
├── Frames（当前栈帧）
│   ├── main.(*App).ProcessImage
│   └── ...
├── Variables（当前作用域的变量）
│   ├── path = "photo.jpg"
│   └── result = nil
├── Goroutines（所有 goroutine）
│   ├── [1] main.main (Running)
│   ├── [12] runtime.gopark (Running)
│   └── [15] main.(*App).BatchProcess.func1 (Debugging) ← 当前
└── Threads
```

当调试 goroutine 时，`F8` 步过只走当前 goroutine。如果想切到另一个 goroutine，在 Goroutines 列表双击即可。

## 查看变量值的几种方式

```go
// 调试时 Alt+F8 计算表达式
a.ctx                          // 看 context
reflect.TypeOf(variable)       // 看变量实际类型
len(resultSlice)               // 看切片长度
unsafe.Sizeof(data)            // 看结构体大小

// 在 Variables 面板可以直接展开 map、slice、struct
```

## 日志断点

有时候代码走到某个位置太多（比如循环里），你不想每次都停，但想看值：

```
右键断点 → More → 取消 Suspend → 勾选 Log message
输入: "处理图片: " + path + ", 结果: " + result
```

这样不会停，但在 Debug Console 会打出日志。

## 远程调试

如果是 Linux 服务器上跑的 Wails 应用：
```bash
# 服务器上编译 debug 版
go build -gcflags="all=-N -l" -o app

# 服务器上运行 headless 版本（如果支持）
dlv exec --headless --listen=:2345 --api-version=2 ./app
```

```text
本地 GoLand → Run → Edit Configurations → Go Remote
Host: 服务器 IP
Port: 2345
```

## 技巧总结

1. **条件断点**比手动 break 更干净
2. **日志断点**适合不想停但又想看值的情况
3. **变量面板拖动变量到编辑器**可以快速查看复杂表达式
4. **`Alt+Click`** 在表达式上可以快速添加 Watch
5. **`F9` 到下一个断点**比一个个 `F8` 快
