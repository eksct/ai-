## 前后端通信的两条"管道"

Wails 里前后端通信有两条独立的管道，很多人混着用：

| 方式 | 方向 | 适用场景 |
|------|------|----------|
| **Bind（方法绑定）** | 前端 → 后端（请求-响应） | CRUD、查询、写入等"调用即返回"的操作 |
| **Events（事件）** | 双向推送 | 进度反馈、通知、状态同步 |

简单说：Bind 是 HTTP 请求那样"你调我，我等结果"；Events 是 WebSocket 那样"你发我收，不管对方在不在线"。

## 你的图像管理 App 用哪个？

```
场景                          选择
────────────────────────────────────────────
用户点击"打开图片"             Bind
读取图片 EXIF 元数据           Bind
批量缩略图生成进度反馈          Events（Go → 前端推送进度）
用户选择标签筛选图片            Bind
后端检测到新文件（文件监听）    Events（Go → 前端通知刷新）
前端让 Go 后台处理大图          Bind（异步）+ Events（进度）
用户点了关闭按钮                Events（前端 → Go 保存状态）
```

## Bind 的数据流

```
前端:
  const result = await Greet("World")
       │
       ▼
wailsjs 生成的胶水代码（JSON 序列化）
       │
       ▼
WebView → Go Runtime → 找到绑定的方法 → 执行
       │
       ▼
Go 返回值 → JSON 序列化 → WebView → Promise resolve
       │
       ▼
前端拿到 result
```

重要：Bind 方法是**同步在 Go 侧执行**的（不启动新 goroutine）。如果你的方法需要跑很长时间，必须在 Go 里手动 `go func()`，然后通过 Events 通知前端结果。
