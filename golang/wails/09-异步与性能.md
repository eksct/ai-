# 异步与性能

## 概述

桌面端不同于 Web 后端——操作卡 100ms 用户就觉得"卡"。图像处理又是计算密集型，处理不当界面直接冻住。核心原则：**不要在 Bind 方法里做耗时操作**。

## Bind 方法默认是同步的

```go
// ❌ 这样会直接卡住前端 10 秒
func (a *App) ProcessBigImage(path string) string {
    time.Sleep(10 * time.Second)  // 模拟处理大图
    return "done"
}
```

用户点击处理按钮，界面像死了一样 10 秒，99% 的用户会直接关掉。

## 正确的异步模式

```go
// ✅ Bind 方法立刻返回，用 Events 通知结果
func (a *App) ProcessBigImage(path string) {
    go func() {
        result := a.doHeavyProcessing(path)
        // 通过 Events 推送给前端
        runtime.EventsEmit(a.ctx, "image:processed", result)
    }()
}
```

前端：
```js
// Bind 调用后立刻返回，不阻塞
ProcessBigImage("photo.jpg");

// 监听处理完成事件
EventsOn("image:processed", (result) => {
    // 更新 UI
});
```

## 带进度的异步任务

图像批量处理时用户需要看到进度，不然他会觉得"是不是卡死了"。

```go
func (a *App) BatchProcess(images []string) {
    a.taskRunning.Store(true)
    a.taskCancelled.Store(false)

    go func() {
        for i, img := range images {
            // 检查是否被取消
            if a.taskCancelled.Load() {
                runtime.EventsEmit(a.ctx, "task:cancelled", nil)
                a.taskRunning.Store(false)
                return
            }

            result := a.processOne(img)

            runtime.EventsEmit(a.ctx, "task:progress", BatchProgress{
                Current:    i + 1,
                Total:      len(images),
                CurrentImg: filepath.Base(img),
                Result:     result,
            })
        }
        runtime.EventsEmit(a.ctx, "task:done", nil)
        a.taskRunning.Store(false)
    }()
}

// 取消任务
func (a *App) CancelBatch() {
    a.taskCancelled.Store(true)
}
```

### 前端进度条

```vue
<template>
  <div v-if="showProgress">
    <progress :value="progress.current" :max="progress.total"></progress>
    <span>{{ progress.current }} / {{ progress.total }}</span>
    <span>{{ progress.currentImg }}</span>
    <button @click="cancel">取消</button>
  </div>
</template>

<script setup>
import { ref, onMounted, onUnmounted } from 'vue'
import { EventsOn, EventsOff } from '../wailsjs/runtime'
import { CancelBatch } from '../wailsjs/go/main/App'

const progress = ref({ current: 0, total: 0, currentImg: '' })
const showProgress = ref(false)

onMounted(() => {
    EventsOn("task:progress", (data) => {
        showProgress.value = true
        progress.value = data
    })
    EventsOn("task:done", () => {
        showProgress.value = false
        // 刷新列表
    })
    EventsOn("task:cancelled", () => {
        showProgress.value = false
    })
})

onUnmounted(() => {
    EventsOff("task:progress")
    EventsOff("task:done")
    EventsOff("task:cancelled")
})

function cancel() {
    CancelBatch()
}
</script>
```

## 图像解码内存控制

### 潜在问题

```go
// ❌ 用户选了一张 8000×6000 的 RAW 图，直接解码
img, _, err := image.Decode(file)  // 可能 OOM
```

### 解决方案

```go
// ✅ 先解码头部，获取尺寸信息
func GetImageInfo(path string) (*ImageInfo, error) {
    f, err := os.Open(path)
    if err != nil {
        return nil, err
    }
    defer f.Close()

    // 只解码头部，不加载像素数据
    cfg, _, err := image.DecodeConfig(f)
    if err != nil {
        return nil, err
    }

    return &ImageInfo{
        Width:  cfg.Width,
        Height: cfg.Height,
        Format: detectFormat(path),
    }, nil
}

// ✅ 缩略图：缩放到合理尺寸再解码
func DecodeThumbnail(path string, maxSize int) (image.Image, error) {
    f, err := os.Open(path)
    if err != nil {
        return nil, err
    }
    defer f.Close()

    src, _, err := image.Decode(f)
    if err != nil {
        return nil, err
    }

    bounds := src.Bounds()
    ratio := float64(maxSize) / float64(bounds.Max.X)
    if ratio > 1 {
        ratio = 1
    }

    newWidth := int(float64(bounds.Max.X) * ratio)
    newHeight := int(float64(bounds.Max.Y) * ratio)

    // 缩放到目标尺寸
    dst := image.NewRGBA(image.Rect(0, 0, newWidth, newHeight))
    draw.BiLinear.Scale(dst, dst.Bounds(), src, src.Bounds(), draw.Over, nil)
    return dst, nil
}
```

## 文件防抖（监听目录变化）

当你用 `fsnotify` 监听图片目录时，复制一个大文件会触发多次事件：

```go
type Debouncer struct {
    timer  *time.Timer
    window time.Duration
}

func NewDebouncer(window time.Duration) *Debouncer {
    return &Debouncer{window: window}
}

func (d *Debouncer) Debounce(key string, fn func()) {
    if d.timer != nil {
        d.timer.Stop()
    }
    d.timer = time.AfterFunc(d.window, fn)
}

// 使用
debouncer := NewDebouncer(300 * time.Millisecond)
watcher, _ := fsnotify.NewWatcher()

go func() {
    for event := range watcher.Events {
        if event.Has(fsnotify.Create) {
            debouncer.Debounce(event.Name, func() {
                runtime.EventsEmit(a.ctx, "file:new", event.Name)
            })
        }
    }
}()
```

## 缩略图缓存

图像管理 App 最频繁的操作就是"浏览目录"，每次都解码原图性能不能接受。

```go
type ThumbnailCache struct {
    dir     string
    cache   sync.Map
    maxSize int
}

func NewThumbnailCache(cacheDir string, maxSize int) *ThumbnailCache {
    os.MkdirAll(cacheDir, 0755)
    return &ThumbnailCache{dir: cacheDir, maxSize: maxSize}
}

func (tc *ThumbnailCache) Get(path string) (string, error) {
    // key 基于文件路径和修改时间
    stat, _ := os.Stat(path)
    key := fmt.Sprintf("%x_%x", path, stat.ModTime().Unix())
    cachePath := filepath.Join(tc.dir, key+".jpg")

    // 缓存存在直接返回
    if _, err := os.Stat(cachePath); err == nil {
        return cachePath, nil
    }

    // 生成缩略图
    tc.generateThumbnail(path, cachePath)
    return cachePath, nil
}
```

## 踩坑记录

### 1. 大量 goroutine 导致界面卡顿

```go
// ❌ 100 张图开 100 个 goroutine 同时处理
for _, img := range images {
    go processOne(img)  // 磁盘 IO 打满，CPU 也打满
}
```

**解决：** 用 worker pool 限制并发数
```go
sem := make(chan struct{}, 4) // 最多 4 个并发
for _, img := range images {
    sem <- struct{}{}
    go func(img string) {
        defer func() { <-sem }()
        processOne(img)
    }(img)
}
```

### 2. Events 发送频率太高

**现象：** 处理 1000 张图，每张图发一次 Events，前端渲染不过来

**解决：** 节流——每 100ms 合并一次进度更新
```go
func (a *App) BatchProcess(images []string) {
    go func() {
        ticker := time.NewTicker(100 * time.Millisecond)
        defer ticker.Stop()

        var lastSent time.Time
        for i, img := range images {
            a.results[i] = a.processOne(img)
            
            if time.Since(lastSent) > 100*time.Millisecond {
                runtime.EventsEmit(a.ctx, "batch:progress", map[string]int{
                    "current": i + 1, "total": len(images),
                })
                lastSent = time.Now()
            }
        }
        // 最后确保发一次完成事件
        runtime.EventsEmit(a.ctx, "batch:done", a.results)
    }()
}
```
