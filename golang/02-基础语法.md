# 基础语法

## 设计原理

### 为什么 Go 的语法是这样

Go 的语言设计在"易读"和"简洁"之间极端偏向易读。它宁愿你多打几个字，也不让你写出看不懂的代码。

```
C 的 i++ 是表达式，可以用在 if 条件里：if (i++ > 0) → 易读？不。
Go 的 i++ 是语句，不能用在表达式里 → 少了一个出 bug 的场景。

Java 有隐式类型转换：long l = intVal → 方便，但可能精度丢失。
Go 没有隐式转换：float64(i) → 啰嗦，但永远不会搞混类型。

GO 变量声明了必须用 → 烦人（很多人骂这个），但代码里永远不会有死变量。
```

### 零值设计的意义

Go 没有构造函数，变量声明后自动有零值。这个设计让 Go 没有一个经典的"对象未初始化"场景。

```go
// Java 里面：
User user;  // null
user.getName();  // NullPointerException!

// Go 里面：
var user User  // 零值，所有字段都是零值
user.Name      // ""（空字符串，不是 nil，不会崩溃）
```

但零值不是万能的——map、slice、chan 的零值是 nil，直接写会 panic。这就是为什么 `var m map[string]int` 之后需要 `make`。

## 核心概念

### 变量声明选择

```go
// 1. var 声明——适合包级变量
var Version = "1.0.0"

// 2. 短变量声明——适合函数内（90% 场景）
name := "Alice"

// 3. 显式类型——适合初始化值不明显时
var count int  // 默认是 0，读代码的人不用猜

// 4. 分组声明——适合相关变量
var (
    timeout = 30 * time.Second
    retries = 3
    debug   = false
)
```

### 常量和 iota

```go
const (
    // iota 从 0 开始递增
    StatusPending = iota  // 0
    StatusApproved        // 1
    StatusRejected        // 2

    // 跳过值
    _ = iota       // 0
    KB = 1 << (10 * iota)  // 1 << 10 = 1024
    MB                    // 1 << 20
    GB                    // 1 << 30
)
```

**生产陷阱：** iota 值依赖常量定义顺序。如果在中间插了一行，所有后续值都变。团队协作时最好显式写值。

## 生产实践

### 命名规范（不是风格，是共识）

```go
// ✅ 好
var userID int        // 驼峰，缩写全大写
func GetUser(id int)  // 不写 GetUserById（Go 的函数名应该短）
var done chan struct{} // 用 struct{} 表示"信号"

// ❌ 不好（违反了 Go 社区的隐性共识）
var userId int        // 缩写不大写
func GetUserById()    // Java 命名迁移
var flag bool = true  // 变量名没表达含义
```

### 变量作用域的陷阱

```go
// 陷阱 1：if 作用域内的变量遮蔽
err := doSomething()     // 包级 err？新声明的？
if err := step1(); err != nil {  // 这里的 err 是新变量
    // step1 的 err
}
// 外面的 err 不受影响——容易让人困惑

// 陷阱 2：循环变量
for _, img := range images {
    // 循环体内新声明的变量
    img := img  // 重新声明，防止闭包捕获问题
    go func() {
        process(img)  // 如果没有上面的 img := img，这里全是最后一张
    }()
}
```

### 类型转换的坑

```go
// int 和 int32 是不同类型
var a int = 10
var b int32 = 20
// a = b  // 编译错误！不能隐式转换
a = int(b)  // 显式转换

// string 和 []byte
s := string([]byte{104, 101, 108, 108, 111})  // "hello"
b := []byte("hello")

// rune 和 byte
s := "你好"
len(s)  // 6（UTF-8 编码，一个汉字 3 字节）
len([]rune(s))  // 2（Unicode 字符数）
```
