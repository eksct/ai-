# 内存管理与性能优化

## 栈 vs 堆

Go 编译器决定变量分配在栈还是堆上——不是由 `new` 或 `&` 决定。

```go
// 逃逸分析
func createUser() *User {
    u := User{Name: "Alice"}  // u 会逃逸到堆上
    return &u                 // 返回指针，编译器检测到逃逸
}

func createUser2() User {
    return User{Name: "Alice"}  // 栈上分配，不逃逸
}
```

查看逃逸分析：
```bash
go build -gcflags="-m" main.go
```

## GC（垃圾回收）

Go 使用**并发三色标记清除 GC**，目标是 STW（Stop The World）时间 < 1ms。

### 如何减少 GC 压力

```go
// ❌ 频繁分配
for i := 0; i < 1000; i++ {
    buf := make([]byte, 1024)  // 每次分配新内存
    process(buf)
}

// ✅ 复用
buf := make([]byte, 1024)
for i := 0; i < 1000; i++ {
    process(buf)
}

// ✅ 用 sync.Pool 复用对象
var pool = sync.Pool{
    New: func() interface{} { return &Image{} },
}

img := pool.Get().(*Image)
// 使用 img
pool.Put(img)
```

## 编译器优化

```go
// 边界检查消除
// ❌ Go 编译器会插入边界检查
for i := 0; i < len(s); i++ {
    _ = s[i]
}

// ✅ 编译器会自动消除明显不会越界的检查
_ = s[3:5]
```

## 性能陷阱

### 1. 字符串拼接

```go
// ❌ 大量拼接
s := ""
for i := 0; i < 1000; i++ {
    s += "a"  // 每次创建新字符串
}

// ✅ strings.Builder
var b strings.Builder
b.Grow(1000)  // 预分配
for i := 0; i < 1000; i++ {
    b.WriteString("a")
}
s := b.String()
```

### 2. 接口装箱

```go
// ❌ 每次传 int 给 interface{} 都会装箱（堆分配）
func print(v interface{}) { fmt.Println(v) }
for i := 0; i < 1000; i++ {
    print(i)  // 1000 次堆分配
}

// ✅ 用具体类型
func printInt(v int) { fmt.Println(v) }
```

### 3. 循环内分配

```go
// ❌ 循环内声明导致多次分配
for i := 0; i < 1000; i++ {
    user := User{Name: "Alice"}  // 1000 次分配
    process(user)
}

// ✅ 提到循环外
user := User{}
for i := 0; i < 1000; i++ {
    user.Name = "Alice"
    process(user)
}
```

### 4. defer 性能

```go
// 高频函数中 defer 有微小开销
// 如果每秒调用百万次才需要考虑
// 一般业务代码完全不用管
```

## 并发性能

```go
// GOMAXPROCS — 默认等于 CPU 核数
// 不是越多 goroutine 越快

// 限制并发数
sem := make(chan struct{}, 4)  // 最多 4 个并发
for _, img := range images {
    sem <- struct{}{}
    go func(img string) {
        defer func() { <-sem }()
        process(img)
    }(img)
}
for i := 0; i < cap(sem); i++ {
    sem <- struct{}{}  // 等待所有完成
}
```
