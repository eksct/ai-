# 泛型

## 设计原理

### Go 为什么迟迟不加泛型

```
Go 1.0 ~ 1.17：没有泛型（约 10 年）
Go 1.18：终于加入泛型
```

不是 Go 团队懒，而是他们在**编译速度和泛型表达力之间做权衡**：

```
C++ 模板：编译慢、错误信息难读、但表达力极强
Java 泛型：类型擦除、运行时无泛型信息、但兼容性好
Go 泛型：基于类型约束、编译时展开、对编译速度影响小
```

Go 的选择是"够用就行"——你可以写泛型函数和泛型类型，但不能做模板元编程、不能做特化、不能做变长模板参数。

### 和 Java 泛型的对比

```java
// Java 泛型是"假"的（类型擦除）
List<String> list = new ArrayList<>();
// 编译后变成：List list = new ArrayList();
// 运行时不知道 List 里面装的是什么类型
```

```go
// Go 泛型是"真"的（编译时展开）
type Stack[T any] struct { ... }
// 编译后：Stack_int、Stack_string 各生成一份代码
```

**GO 泛型的代价：** 编译二进制变大（每个实例化类型生成一份代码）。如果你的泛型类型有 10 种不同的实例化类型，编译出 10 份代码。

## 核心概念

### 类型约束

```go
// Go 1.18 引入的约束写法
type Number interface {
    ~int | ~int8 | ~int16 | ~int32 | ~int64 |
    ~float32 | ~float64
}

// ~ 的意思是"以此类型为底层类型的都算"
type MyInt int  // 底层类型是 int，满足 ~int

// 内置约束
constrains.Integer    // 所有整数类型
constrains.Float      // 所有浮点类型
constrains.Complex    // 所有复数类型
constrains.Ordered    // 所有可排序类型（整数、浮点、字符串）
```

### 泛型函数

```go
// 非泛型：每种类型写一个
func MaxInt(a, b int) int { if a > b { return a }; return b }
func MaxFloat(a, b float64) float64 { if a > b { return a }; return b }
func MaxString(a, b string) string { if a > b { return a }; return b }

// 泛型：一个搞定
func Max[T constraints.Ordered](a, b T) T {
    if a > b { return a }
    return b
}
```

### 泛型类型

```go
type Result[T any] struct {
    Value T
    Err   error
}

func Success[T any](value T) Result[T] {
    return Result[T]{Value: value}
}

func Failure[T any](err error) Result[T] {
    return Result[T]{Err: err}
}

// 使用
result := Success[int](42)
if result.Err == nil {
    fmt.Println(result.Value)
}
```

## 生产实践

### 什么时候用泛型

```
✅ 适合用：数据容器（Stack、Queue、List、Tree）
✅ 适合用：算法（排序、查找、过滤、映射）
✅ 适合用：通用工具函数（Max、Min、Contains）
✅ 适合用：Result/Optional 模式

❌ 不适合：业务逻辑
❌ 不适合：只有一个类型参数的抽象（不如直接写具体类型）
❌ 不适合：性能敏感路径（增加编译和运行时开销）
```

### 泛型不能做的事情

```go
// 1. 不能做类型断言
func Print[T any](v T) {
    // if v.(string) { }  // 编译错误！泛型类型不能断言
    // 用反射或者类型约束
}

// 2. 不能做运算
func Add[T any](a, b T) T {
    // return a + b  // 编译错误！不是所有类型都能 +
    // 用 constraints.Ordered 约束实现
}

// 3. 不能用在 switch type 中（可以用约束替代）
```

### 真实案例：图像处理中的泛型

```go
// 缓存的泛型实现
type Cache[K comparable, V any] struct {
    mu    sync.RWMutex
    items map[K]V
}

func NewCache[K comparable, V any](initial map[K]V) *Cache[K, V] {
    if initial == nil {
        initial = make(map[K]V)
    }
    return &Cache[K, V]{items: initial}
}

func (c *Cache[K, V]) Get(key K) (V, bool) {
    c.mu.RLock()
    defer c.mu.RUnlock()
    v, ok := c.items[key]
    return v, ok
}

func (c *Cache[K, V]) Set(key K, value V) {
    c.mu.Lock()
    defer c.mu.Unlock()
    c.items[key] = value
}

// 不同类型的缓存
thumbnailCache := NewCache[string, *Image](nil)
metadataCache := NewCache[string, *Metadata](nil)
configCache := NewCache[string, string](nil)
```
