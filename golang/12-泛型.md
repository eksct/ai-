# 泛型（Go 1.18+）

## 概述

Go 1.18 引入了泛型——这是 Go 历史上最大的语法变更。泛型让你写一个函数/类型可以处理不同类型的数据。

## 泛型函数

```go
// 非泛型版本：每种类型写一个
func SumInts(m map[string]int64) int64 {
    var s int64
    for _, v := range m { s += v }
    return s
}

func SumFloats(m map[string]float64) float64 {
    var s float64
    for _, v := range m { s += v }
    return s
}

// 泛型版本：一个函数搞定
func Sum[K comparable, V int64 | float64](m map[K]V) V {
    var s V
    for _, v := range m { s += v }
    return s
}
```

## 泛型类型

```go
// 泛型栈
type Stack[T any] struct {
    items []T
}

func (s *Stack[T]) Push(item T) {
    s.items = append(s.items, item)
}

func (s *Stack[T]) Pop() (T, bool) {
    if len(s.items) == 0 {
        var zero T
        return zero, false
    }
    item := s.items[len(s.items)-1]
    s.items = s.items[:len(s.items)-1]
    return item, true
}

func (s *Stack[T]) IsEmpty() bool {
    return len(s.items) == 0
}

// 使用
intStack := Stack[int]{}
intStack.Push(1)
intStack.Push(2)
val, _ := intStack.Pop()
fmt.Println(val)  // 2
```

## 类型约束

```go
// 内置约束（constraints 包已废弃，用 builtin）
type Number interface {
    ~int | ~int8 | ~int16 | ~int32 | ~int64 |
    ~uint | ~uint8 | ~uint16 | ~uint32 | ~uint64 |
    ~float32 | ~float64
}

// ~ 表示允许基于该类型的自定义类型
type MyInt int  // ~int 允许，int 不允许

// 使用约束
func Min[T Number](a, b T) T {
    if a < b { return a }
    return b
}

// comparable 是内置约束（支持 == 和 != 的类型）
func Contains[T comparable](s []T, target T) bool {
    for _, v := range s {
        if v == target {
            return true
        }
    }
    return false
}

// any 是 interface{} 的别名
func Print[T any](v T) {
    fmt.Println(v)
}
```

## 应用场景

### 集合工具

```go
func Filter[T any](s []T, fn func(T) bool) []T {
    var result []T
    for _, v := range s {
        if fn(v) {
            result = append(result, v)
        }
    }
    return result
}

func Map[T, U any](s []T, fn func(T) U) []U {
    result := make([]U, len(s))
    for i, v := range s {
        result[i] = fn(v)
    }
    return result
}

// 使用
numbers := []int{1, 2, 3, 4, 5}
doubled := Map(numbers, func(x int) int { return x * 2 })
even := Filter(numbers, func(x int) bool { return x%2 == 0 })
```

### Optional 模式

```go
type Option[T any] struct {
    value T
    ok    bool
}

func Some[T any](value T) Option[T] {
    return Option[T]{value: value, ok: true}
}

func None[T any]() Option[T] {
    return Option[T]{ok: false}
}

func (o Option[T]) IsPresent() bool { return o.ok }

func (o Option[T]) OrElse(defaultVal T) T {
    if o.ok { return o.value }
    return defaultVal
}
```

## 泛型在 Wails 项目中的使用

图像管理中可能需要通用的缓存：

```go
type Cache[K comparable, V any] struct {
    mu    sync.RWMutex
    data  map[K]V
    ttl   time.Duration
}

func NewCache[K comparable, V any](ttl time.Duration) *Cache[K, V] {
    return &Cache[K, V]{
        data: make(map[K]V),
        ttl:  ttl,
    }
}

func (c *Cache[K, V]) Get(key K) (V, bool) {
    c.mu.RLock()
    defer c.mu.RUnlock()
    v, ok := c.data[key]
    return v, ok
}

func (c *Cache[K, V]) Set(key K, value V) {
    c.mu.Lock()
    defer c.mu.Unlock()
    c.data[key] = value
}

// 使用
thumbnailCache := NewCache[string, *Image]()
thumbnailCache.Set("photo.jpg", thumbnail)
```

## 什么时候用泛型

```go
// ✅ 适合：集合工具函数
// Filter, Map, Reduce, Contains 等

// ✅ 适合：通用数据结构
// Stack, Queue, Cache, Result 等

// ❌ 不适合：业务逻辑
// 业务代码里泛型反而降低可读性

// ❌ 不适合：简单的类型替换
// 如果只有一种类型，不需要泛型
```

**经验：** 第一次写泛型时容易过度使用。"能用"不意味着"应该用"。如果泛型让代码更难理解，就用具体类型。
