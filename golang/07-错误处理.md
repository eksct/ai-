# 错误处理

## 概述

Go 的错误处理是显式的——错误作为返回值，调用方必须处理（或者显式忽略）。这和 Java 的 try-catch 完全相反的设计哲学。

## error 接口

```go
type error interface {
    Error() string
}
```

任何实现了 `Error() string` 方法的类型都是 error。

## 基本用法

```go
// 创建错误
errors.New("something went wrong")
fmt.Errorf("user %d not found", id)

// 返回错误（标准模式）
func GetUser(id int) (*User, error) {
    if id <= 0 {
        return nil, errors.New("invalid user id")
    }
    return &User{ID: id}, nil
}

// 调用方处理
user, err := GetUser(42)
if err != nil {
    log.Printf("获取用户失败: %v", err)
    return
}
fmt.Println(user.Name)
```

**最常见的 Go 代码模式：**
```go
result, err := doSomething()
if err != nil {
    // 处理错误
    return
}
// 继续使用 result
```

## 自定义错误类型

```go
type AppError struct {
    Code    int
    Message string
    Err     error  // 原始错误（包装用）
}

func (e *AppError) Error() string {
    return fmt.Sprintf("code=%d: %s", e.Code, e.Message)
}

func (e *AppError) Unwrap() error {
    return e.Err
}

// 使用
if err != nil {
    return &AppError{
        Code:    500,
        Message: "处理失败",
        Err:     err,
    }
}
```

## 错误包装

```go
import "fmt"

// Go 1.13+ 用 %w 包装错误
func ReadConfig(path string) error {
    data, err := os.ReadFile(path)
    if err != nil {
        return fmt.Errorf("读取配置失败 %s: %w", path, err)
    }
    // ...
}

// 解包
err := ReadConfig("/etc/app.conf")
if errors.Is(err, os.ErrNotExist) {
    fmt.Println("配置文件不存在")
}

// 检查特定错误类型
var appErr *AppError
if errors.As(err, &appErr) {
    fmt.Println("错误码:", appErr.Code)
}
```

### errors.Is 和 errors.As

```go
// errors.Is：判断错误链中是否包含某个错误（值比较）
if errors.Is(err, os.ErrNotExist) {
    // 文件不存在
}

// errors.As：判断错误链中是否有匹配的类型（赋值）
var pathErr *os.PathError
if errors.As(err, &pathErr) {
    fmt.Println("路径:", pathErr.Path)
}
```

## 错误处理最佳实践

### 1. 不要忽略错误

```go
// ❌ 不要这样
os.Remove("temp.txt")  // 忽略错误

// ✅ 至少 log
if err := os.Remove("temp.txt"); err != nil {
    log.Printf("删除临时文件失败: %v", err)
}
```

### 2. 不要重复处理错误

```go
// ❌ 既 log 又 return
func do() error {
    err := step1()
    if err != nil {
        log.Printf("step1 failed: %v", err)  // 在这里 log
        return fmt.Errorf("do failed: %w", err)  // 又在调用方 log
    }
}
```

通常：**底层 log，或者上层 log，不要重复**。下层暴露错误，上层决定是否要 log。

### 3. 区分"可恢复"和"不可恢复"

```go
// 可恢复：调用方决定怎么处理
result, err := doSomething()
if err != nil {
    // 可以重试、用默认值、返回 fallback
    result = defaultValue
}

// 不可恢复：程序不能继续
if err := initDatabase(); err != nil {
    log.Fatalf("数据库初始化失败: %v", err)
    // 或 panic(err)
}
```

### 4. sentinel error（哨兵错误）

```go
var (
    ErrNotFound = errors.New("not found")
    ErrForbidden = errors.New("forbidden")
)

func GetUser(id int) (*User, error) {
    return nil, ErrNotFound
}

// 调用方
if errors.Is(err, ErrNotFound) {
    // 处理"未找到"的情况
}
```

**注意：** sentinel error 本质上是全局变量，包外可以直接引用。如果怕被修改，用函数返回（如 `os.ErrNotExist` 就是变量）。

## panic 和 recover

```go
// panic：不可恢复的错误
func must(err error) {
    if err != nil {
        panic(err)
    }
}

// recover：捕获 panic
func safeCall() {
    defer func() {
        if r := recover(); r != nil {
            log.Printf("recovered: %v", r)
        }
    }()
    dangerousCall()  // 如果 panic，被上面的 recover 捕获
}
```

## 在 Wails 中处理错误

```go
// Bind 方法返回 error，Wails 会把 error 转为前端的 rejected Promise
func (a *App) LoadImage(path string) (*Image, error) {
    if path == "" {
        return nil, fmt.Errorf("路径不能为空")
    }

    file, err := os.Open(path)
    if err != nil {
        return nil, fmt.Errorf("打开文件失败: %w", err)
    }
    defer file.Close()

    // 解码图片...
    img, err := decodeImage(file)
    if err != nil {
        return nil, fmt.Errorf("图片解码失败: %w", err)
    }

    return img, nil
}
```

前端：
```js
App.LoadImage("test.jpg")
    .then(img => { /* 显示图片 */ })
    .catch(err => { /* 显示错误提示 */ });
```
