# 生产集群搭建

## 部署方案对比

| 方案 | 用途 | 难易度 | 适用场景 |
|------|------|--------|----------|
| **Minikube** | 本地开发 | ⭐ | 个人学习、开发调试 |
| **kubeadm** | 生产部署 | ⭐⭐⭐ | 自建集群 |
| **Kubespray** | Ansible 部署 | ⭐⭐⭐ | 自动化部署 |
| **Kops** | AWS 集群 | ⭐⭐ | AWS 环境 |
| **Rancher** | 企业级管理 | ⭐⭐ | 多集群管理 |
| **K3s** | 边缘计算 | ⭐ | 资源受限环境 |
| **Kind** | 本地测试 | ⭐ | CI/CD 测试 |

## 本地开发环境

### Minikube

```bash
# 安装 Minikube
choco install minikube
# 或下载 exe：https://minikube.sigs.k8s.io/docs/start/

# 启动集群
minikube start --driver=docker
minikube start --cpus=4 --memory=8g --disk-size=50g

# 查看状态
minikube status
minikube ip

# 开启仪表盘
minikube dashboard

# 安装插件
minikube addons enable ingress
minikube addons enable metrics-server
minikube addons enable metallb

# 停止/删除
minikube stop
minikube delete
```

### Kind（Kubernetes in Docker）

```bash
# 安装
choco install kind
# 或 go install sigs.k8s.io/kind@latest

# 创建集群
kind create cluster --name dev

# 多节点集群配置
cat <<EOF | kind create cluster --config=-
kind: Cluster
apiVersion: kind.x-k8s.io/v1alpha4
nodes:
  - role: control-plane
  - role: worker
  - role: worker
EOF

# 加载镜像
kind load docker-image my-app:v1 --name dev

# 删除
kind delete cluster --name dev
```

## kubeadm 生产部署

### 架构规划

```
机器规格建议：
  Control Plane: 4C 8G SSD 100G × 3（奇数台，Raft 投票）
  Worker Node:   8C 32G SSD 200G × N（根据负载扩展）

网络规划：
  Node Network:     192.168.1.0/24
  Pod CIDR:         10.244.0.0/16（Calico）
  Service CIDR:     10.96.0.0/12
```

### 前置条件（所有节点）

```bash
# 1. 安装容器运行时（containerd）
cat <<EOF | sudo tee /etc/modules-load.d/containerd.conf
overlay
br_netfilter
EOF

sudo modprobe overlay
sudo modprobe br_netfilter

cat <<EOF | sudo tee /etc/sysctl.d/99-kubernetes.conf
net.bridge.bridge-nf-call-iptables  = 1
net.bridge.bridge-nf-call-ip6tables = 1
net.ipv4.ip_forward                 = 1
EOF

sudo sysctl --system

# 2. 安装 containerd
sudo apt update
sudo apt install -y containerd

sudo mkdir -p /etc/containerd
containerd config default | sudo tee /etc/containerd/config.toml

# 修改配置启用 SystemdCgroup
sudo sed -i 's/SystemdCgroup = false/SystemdCgroup = true/' /etc/containerd/config.toml

sudo systemctl restart containerd
sudo systemctl enable containerd

# 3. 关闭 swap
sudo swapoff -a
sudo sed -i '/ swap / s/^\(.*\)$/#\1/g' /etc/fstab

# 4. 安装 kubeadm, kubelet, kubectl
curl -fsSL https://pkgs.k8s.io/core:/stable:/v1.36/deb/Release.key | sudo gpg --dearmor -o /etc/apt/keyrings/kubernetes-apt-keyring.gpg
echo 'deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] https://pkgs.k8s.io/core:/stable:/v1.36/deb/ /' | sudo tee /etc/apt/sources.list.d/kubernetes.list

sudo apt update
sudo apt install -y kubelet kubeadm kubectl
sudo apt-mark hold kubelet kubeadm kubectl

# 5. 设置 kubelet 的 cgroup 驱动
cat <<EOF | sudo tee /etc/default/kubelet
KUBELET_EXTRA_ARGS="--cgroup-driver=systemd"
EOF
```

### 初始化 Control Plane

```bash
# 在第一个 Master 节点执行

# 创建配置文件
cat <<EOF > kubeadm-config.yaml
apiVersion: kubeadm.k8s.io/v1beta3
kind: InitConfiguration
localAPIEndpoint:
  advertiseAddress: 192.168.1.10  # 本机 IP
  bindPort: 6443
---
apiVersion: kubeadm.k8s.io/v1beta3
kind: ClusterConfiguration
kubernetesVersion: v1.36.0
controlPlaneEndpoint: "192.168.1.100:6443"  # VIP 或 LB 地址
networking:
  serviceSubnet: "10.96.0.0/12"
  podSubnet: "10.244.0.0/16"
  dnsDomain: "cluster.local"
---
apiVersion: kubeproxy.config.k8s.io/v1alpha1
kind: KubeProxyConfiguration
mode: ipvs             # 推荐使用 IPVS
---
apiVersion: kubelet.config.k8s.io/v1beta1
kind: KubeletConfiguration
cgroupDriver: systemd
EOF

# 初始化
sudo kubeadm init --config=kubeadm-config.yaml --upload-certs

# 配置 kubectl
mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config

# 记录加入命令（后续添加节点用）
# kubeadm join 192.168.1.100:6443 --token <token> --discovery-token-ca-cert-hash <hash>
# kubeadm join 192.168.1.100:6443 --token <token> --discovery-token-ca-cert-hash <hash> --control-plane --certificate-key <key>
```

### 安装 CNI（Calico）

```bash
# 安装 Tigera Operator
kubectl create -f https://raw.githubusercontent.com/projectcalico/calico/v3.27/manifests/tigera-operator.yaml

# 配置 Calico
cat <<EOF | kubectl apply -f -
apiVersion: operator.tigera.io/v1
kind: Installation
metadata:
  name: default
spec:
  calicoNetwork:
    ipPools:
      - cidr: 10.244.0.0/16
        encapsulation: VXLANCrossSubnet
        natOutgoing: true
    nodeAddressAutodetectionV4:
      interface: "eth0"
EOF
```

### 添加 Worker 节点

```bash
# 在 Worker 节点上执行
sudo kubeadm join 192.168.1.100:6443 --token <token> \
  --discovery-token-ca-cert-hash sha256:<hash>
```

### 验证集群

```bash
# 查看节点状态
kubectl get nodes
NAME              STATUS   ROLES           AGE   VERSION
master-01         Ready    control-plane   1h    v1.36.0
master-02         Ready    control-plane   55m   v1.36.0
master-03         Ready    control-plane   50m   v1.36.0
worker-01         Ready    worker          45m   v1.36.0
worker-02         Ready    worker          40m   v1.36.0

# 查看系统 Pod
kubectl get pods -n kube-system

# 运行测试
kubectl run test --image=nginx --port=80
kubectl expose pod test --port=80
kubectl get svc test
```

## 高可用配置

### 控制平面高可用

```
                   LB (VIP / Load Balancer)
                  /          |          \
         Master-01      Master-02      Master-03
         (kube-apiserver)  (kube-apiserver)  (kube-apiserver)
             |                |                |
            etcd ─────────── etcd ─────────── etcd
```

### 使用 HAProxy + Keepalived

```bash
# 在 LB 节点安装
sudo apt install haproxy keepalived

# haproxy.cfg
cat <<EOF | sudo tee /etc/haproxy/haproxy.cfg
frontend k8s-api
    bind *:6443
    mode tcp
    default_backend k8s-masters

backend k8s-masters
    mode tcp
    balance roundrobin
    server master-01 192.168.1.10:6443 check
    server master-02 192.168.1.11:6443 check
    server master-03 192.168.1.12:6443 check
EOF

# keepalived.conf
cat <<EOF | sudo tee /etc/keepalived/keepalived.conf
vrrp_instance VI_1 {
    state MASTER
    interface eth0
    virtual_router_id 51
    priority 100
    advert_int 1
    authentication {
        auth_type PASS
        auth_pass 1234
    }
    virtual_ipaddress {
        192.168.1.100/24
    }
}
EOF
```

## 生产必备组件

完成基础搭建后应立即部署：

```bash
# 1. Metrics Server（监控基础）
kubectl apply -f https://github.com/kubernetes-sigs/metrics-server/releases/latest/download/components.yaml

# 2. Ingress Controller
helm repo add ingress-nginx https://kubernetes.github.io/ingress-nginx
helm upgrade --install ingress-nginx ingress-nginx/ingress-nginx \
  --namespace ingress-nginx --create-namespace \
  --set controller.replicaCount=3 \
  --set controller.service.type=LoadBalancer

# 3. 证书管理（cert-manager）
helm repo add jetstack https://charts.jetstack.io
helm upgrade --install cert-manager jetstack/cert-manager \
  --namespace cert-manager --create-namespace \
  --set installCRDs=true

# 4. 存储类（以 AWS 为例）
kubectl apply -f https://raw.githubusercontent.com/kubernetes-sigs/aws-ebs-csi-driver/master/deploy/manifests/controller.yaml

# 5. 监控（kube-prometheus-stack）
helm upgrade --install monitoring prometheus-community/kube-prometheus-stack \
  --namespace monitoring --create-namespace

# 6. 日志收集（Loki + Promtail）
helm upgrade --install loki grafana/loki --namespace logging --create-namespace
helm upgrade --install promtail grafana/promtail --namespace logging

# 7. 自动扩缩容（KEDA 或 metrics-server 自带的 HPA）
helm upgrade --install keda kedacore/keda --namespace keda --create-namespace
```

## 集群升级

```bash
# 升级 Control Plane（逐个升级）
# 1. 升级 kubeadm
sudo apt update && sudo apt install -y kubeadm=1.36.0-*
sudo kubeadm upgrade plan
sudo kubeadm upgrade apply v1.36.0

# 2. 升级 kubelet
sudo apt install -y kubelet=1.36.0-* kubectl=1.36.0-*
sudo systemctl restart kubelet

# 升级 Worker 节点
sudo kubeadm upgrade node
sudo apt install -y kubelet=1.36.0-* kubectl=1.36.0-*
sudo systemctl restart kubelet
```

## 备份与恢复

```bash
# etcd 备份
sudo ETCDCTL_API=3 etcdctl \
  --cacert=/etc/kubernetes/pki/etcd/ca.crt \
  --cert=/etc/kubernetes/pki/etcd/server.crt \
  --key=/etc/kubernetes/pki/etcd/server.key \
  snapshot save /backup/etcd-snapshot-$(date +%Y%m%d).db

# etcd 恢复
sudo ETCDCTL_API=3 etcdctl snapshot restore /backup/etcd-snapshot.db \
  --data-dir=/var/lib/etcd-restored
```
