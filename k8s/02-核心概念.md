# Kubernetes 核心概念

## Namespace（命名空间）

Namespace 提供虚拟集群隔离，适用于多环境、多团队场景。

```bash
# 查看所有命名空间
kubectl get namespaces

# 创建命名空间
kubectl create namespace dev

# 在指定命名空间操作
kubectl get pods -n dev
kubectl run nginx --image=nginx -n dev

# 切换命名空间上下文
kubectl config set-context --current --namespace=dev
```

**常见内置 Namespace：**
| Namespace | 用途 |
|-----------|------|
| `default` | 默认命名空间 |
| `kube-system` | K8s 系统组件 |
| `kube-public` | 所有用户可读（包括未认证用户） |
| `kube-node-lease` | 节点心跳信息 |

**资源配额示例：**
```yaml
apiVersion: v1
kind: ResourceQuota
metadata:
  name: dev-quota
  namespace: dev
spec:
  hard:
    pods: "10"
    requests.cpu: "4"
    requests.memory: 8Gi
    limits.cpu: "8"
    limits.memory: 16Gi
    persistentvolumeclaims: "5"
    services: "10"
```

## Node（节点）

Node 是 K8s 集群中的工作机器，可以是物理机或虚拟机。

```bash
# 查看节点
kubectl get nodes
kubectl get nodes -o wide          # 详细信息（IP、OS、内核等）
kubectl describe node node-name    # 节点详情

# 节点状态
kubectl top node                   # 查看节点资源使用
```

**节点状态：**
- `Ready`：节点健康，可以调度 Pod
- `NotReady`：节点不健康
- `SchedulingDisabled`：禁止调度（drain 后）

**节点管理：**
```bash
# 维护节点（驱逐 Pod）
kubectl drain node-name --ignore-daemonsets

# 恢复调度
kubectl uncordon node-name

# 标记节点
kubectl label node node-name disktype=ssd
kubectl taint node node-name key=value:NoSchedule
```

## Pod（最小的调度单元）

Pod 是 K8s 中**最小的可部署单元**，一个 Pod 包含一个或多个容器。

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: my-pod
  labels:
    app: my-app
spec:
  containers:
    - name: nginx
      image: nginx:alpine
      ports:
        - containerPort: 80
```

```bash
# 运行 Pod
kubectl run nginx --image=nginx:alpine --port=80
kubectl apply -f pod.yaml

# 查看 Pod
kubectl get pods
kubectl get pods -o wide        # 含 IP 和节点信息
kubectl describe pod nginx
kubectl logs nginx
kubectl logs -f nginx           # 实时日志

# 进入 Pod
kubectl exec -it nginx -- sh
kubectl exec -it nginx -- /bin/bash

# 删除 Pod
kubectl delete pod nginx
```

## Label（标签）和 Selector（选择器）

Label 是附加到 K8s 对象上的键值对，用于标识和组织资源。

```yaml
metadata:
  labels:
    app: nginx
    environment: production
    tier: frontend
    release: v1.2.3
```

```bash
# 标签操作
kubectl label pod nginx env=prod          # 添加标签
kubectl label pod nginx env-              # 删除标签
kubectl label pod nginx env=staging --overwrite  # 覆盖

# 选择器查询
kubectl get pods -l app=nginx
kubectl get pods -l 'environment in (prod, staging)'
kubectl get pods -l '!tier'              # 不包含某个标签
kubectl get pods -l app=nginx,env=prod   # 多个条件（AND）
```

## Annotation（注解）

Annotation 与 Label 类似，但用于存储非标识性的元数据：

```yaml
metadata:
  annotations:
    nginx.ingress.kubernetes.io/proxy-body-size: "10m"
    prometheus.io/scrape: "true"
    prometheus.io/port: "9090"
```

- Label：用于选择和分组（长度限制 63 字符）
- Annotation：用于附加信息（无长度限制），不能被 Selector 选择

## 常用命令速查

```bash
# 集群信息
kubectl cluster-info               # 集群信息
kubectl version                    # 版本
kubectl get componentstatuses      # 组件状态

# 资源管理
kubectl get all                    # 获取 default 下所有资源
kubectl api-resources              # 查看所有支持的资源类型
kubectl explain pod                # 查看资源字段说明

# 别名（强烈推荐设置）
alias k='kubectl'
alias kg='kubectl get'
alias kd='kubectl describe'
alias kdel='kubectl delete'
alias ka='kubectl apply -f'
alias kaf='kubectl apply -f'
alias kl='kubectl logs'
alias kex='kubectl exec -it'
```

## 资源创建方式对比

| 方式 | 命令 | 适用场景 |
|------|------|----------|
| 命令式命令 | `kubectl run nginx --image=nginx` | 快速测试 |
| 命令式对象配置 | `kubectl create -f nginx.yaml` | 简单场景 |
| 声明式对象配置 | `kubectl apply -f nginx.yaml` | **生产推荐** |

> **生产环境推荐使用声明式方式**（`kubectl apply -f`），便于版本管理和 GitOps。
