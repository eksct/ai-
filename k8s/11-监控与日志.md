# 监控与日志

## 监控体系

K8s 监控的三层架构：

```
指标采集层 (Metrics Collection)
     ↓
指标存储层 (Metrics Storage)
     ↓
可视化与告警层 (Visualization & Alerting)
```

### 常用技术栈

```
Prometheus + Grafana（监控）
    ├── Node Exporter（节点指标）
    ├── kube-state-metrics（K8s 对象指标）
    └── cAdvisor（容器指标，内置在 kubelet）

Elasticsearch + Fluentd/Filebeat + Kibana（日志）
    或 Loki + Promtail + Grafana（轻量日志）
```

## Prometheus + Grafana

### 安装

```bash
# 使用 Helm 安装 Prometheus Stack
helm repo add prometheus-community https://prometheus-community.github.io/helm-charts
helm repo update

# 安装 kube-prometheus-stack（包含 Prometheus + Grafana + AlertManager）
helm install monitoring prometheus-community/kube-prometheus-stack \
  --namespace monitoring \
  --create-namespace \
  -f prometheus-values.yaml
```

### 配置

```yaml
# prometheus-values.yaml
prometheus:
  prometheusSpec:
    retention: 15d                     # 数据保留时间
    retentionSize: 50GB                # 存储大小限制
    resources:
      requests:
        memory: 2Gi
        cpu: 500m
      limits:
        memory: 4Gi
        cpu: 1
    storageSpec:
      volumeClaimTemplate:
        spec:
          storageClassName: gp3
          accessModes: ["ReadWriteOnce"]
          resources:
            requests:
              storage: 100Gi
    ruleSelectorNilUsesHelmValues: false
    serviceMonitorSelectorNilUsesHelmValues: false

alertmanager:
  enabled: true
  config:
    global:
      resolve_timeout: 5m
      slack_api_url: 'https://hooks.slack.com/services/xxx'
    route:
      group_by: ['namespace', 'alertname']
      group_wait: 30s
      group_interval: 5m
      repeat_interval: 4h
      receiver: 'slack-notifications'
    receivers:
      - name: 'slack-notifications'
        slack_configs:
          - channel: '#alerts'
            send_resolved: true
            title: '{{ template "slack.title" . }}'
            text: '{{ template "slack.text" . }}'

grafana:
  adminPassword: admin123
  persistence:
    enabled: true
    size: 10Gi
  defaultDashboardsEnabled: true
  ingress:
    enabled: true
    ingressClassName: nginx
    hosts:
      - grafana.example.com
```

### ServiceMonitor（自动发现指标端点）

```yaml
apiVersion: monitoring.coreos.com/v1
kind: ServiceMonitor
metadata:
  name: my-app-monitor
  namespace: monitoring
  labels:
    release: monitoring
spec:
  selector:
    matchLabels:
      app: my-app
  namespaceSelector:
    matchNames:
      - production
  endpoints:
    - port: metrics          # Service 中的端口名
      path: /metrics
      interval: 15s          # 采集间隔
      scrapeTimeout: 10s
  targetLabels:
    - app
```

### 应用暴露指标（自定义指标）

```yaml
apiVersion: v1
kind: Service
metadata:
  name: my-app
  labels:
    app: my-app
  annotations:
    prometheus.io/scrape: "true"
    prometheus.io/port: "8080"
    prometheus.io/path: "/metrics"
spec:
  selector:
    app: my-app
  ports:
    - name: http
      port: 8080
    - name: metrics
      port: 9090
```

## 核心监控指标

### 集群层面

```promql
# 节点 CPU 使用率
100 - (avg by(instance) (rate(node_cpu_seconds_total{mode="idle"}[5m])) * 100)

# 节点内存使用率
(1 - (node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes)) * 100

# 节点磁盘使用率
(node_filesystem_size_bytes{mountpoint="/"} - node_filesystem_free_bytes{mountpoint="/"}) / node_filesystem_size_bytes{mountpoint="/"} * 100
```

### Pod/容器层面

```promql
# Pod CPU 使用率（相对于 request）
rate(container_cpu_usage_seconds_total{container!="", pod=~"my-app.*"}[5m]) / container_spec_cpu_shares{container!=""} * 100

# Pod 内存使用率
container_memory_working_set_bytes{container!="", pod=~"my-app.*"} / container_spec_memory_limit_bytes{container!=""} * 100

# Pod 重启次数
rate(kube_pod_container_status_restarts_total[5m])
```

### 应用层面

```promql
# HTTP 请求速率
rate(http_requests_total{job="my-app"}[5m])

# 请求错误率（4xx/5xx）
rate(http_requests_total{job="my-app", status=~"5.."}[5m]) / rate(http_requests_total{job="my-app"}[5m]) * 100

# P99 延迟
histogram_quantile(0.99, rate(http_request_duration_seconds_bucket{job="my-app"}[5m]))
```

## 告警规则

```yaml
# prometheus-rules.yaml
apiVersion: monitoring.coreos.com/v1
kind: PrometheusRule
metadata:
  name: k8s-alerts
  namespace: monitoring
spec:
  groups:
    - name: kubernetes-pods
      rules:
        - alert: PodHighCPUUsage
          expr: sum by(namespace, pod) (rate(container_cpu_usage_seconds_total{container!=""}[5m])) > 0.8
          for: 5m
          labels:
            severity: warning
          annotations:
            summary: "Pod CPU usage is high"
            description: "Pod {{ $labels.pod }} in {{ $labels.namespace }} CPU usage > 80% for 5m"

        - alert: PodCrashLooping
          expr: increase(kube_pod_container_status_restarts_total[1h]) > 3
          for: 10m
          labels:
            severity: critical
          annotations:
            summary: "Pod is crash looping"
            description: "Pod {{ $labels.pod }} restarted {{ $value }} times in 1h"

        - alert: PodNotReady
          expr: sum by(namespace, pod) (kube_pod_status_phase{phase=~"Pending|Unknown|Failed"}) > 0
          for: 10m
          labels:
            severity: critical
          annotations:
            summary: "Pod is not ready"
            description: "Pod {{ $labels.pod }} is in {{ $labels.phase }} state"

        - alert: PersistentVolumeUsage
          expr: (kubelet_volume_stats_used_bytes / kubelet_volume_stats_capacity_bytes) * 100 > 85
          for: 5m
          labels:
            severity: warning
          annotations:
            summary: "PVC usage is high"
            description: "PVC usage is {{ $value }}%"

    - name: kubernetes-nodes
      rules:
        - alert: NodeNotReady
          expr: kube_node_status_condition{condition="Ready", status="true"} == 0
          for: 5m
          labels:
            severity: critical
          annotations:
            summary: "Node is NotReady"
            description: "Node {{ $labels.node }} is not ready"

        - alert: NodeMemoryPressure
          expr: kube_node_status_condition{condition="MemoryPressure", status="true"} == 1
          for: 10m
          labels:
            severity: warning
          annotations:
            summary: "Node memory pressure"
            description: "Node {{ $labels.node }} has memory pressure"
```

## 日志收集

### EFK 架构（Elasticsearch + Fluentd + Kibana）

```bash
# 安装 Elasticsearch
helm install elasticsearch elastic/elasticsearch \
  --namespace logging \
  --create-namespace \
  --set replicas=3

# 安装 Kibana
helm install kibana elastic/kibana \
  --namespace logging \
  --set service.type=NodePort

# 安装 Fluentd
helm install fluentd stable/fluentd \
  --namespace logging \
  -f fluentd-values.yaml
```

### Loki + Promtail + Grafana

更适合与 Prometheus 配合的轻量方案：

```bash
helm repo add grafana https://grafana.github.io/helm-charts
helm repo update

helm install loki grafana/loki \
  --namespace logging \
  --create-namespace

helm install promtail grafana/promtail \
  --namespace logging \
  --set config.lokiAddress=http://loki:3100/loki/api/v1/push
```

### 容器日志标准

```yaml
# 应用日志应该输出到 stdout/stderr
# 不要写到文件
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  template:
    spec:
      containers:
        - name: app
          image: my-app:latest
          # 应用日志输出到 stdout/stderr
          # kubectl logs 会自动收集
```

```bash
# 查看日志
kubectl logs pod/my-app
kubectl logs deployment/my-app          # 查看所有副本
kubectl logs -l app=my-app              # 标签选择

# 指定容器（多容器 Pod）
kubectl logs pod/my-app -c sidecar

# 实时跟踪
kubectl logs -f pod/my-app

# 时间范围
kubectl logs pod/my-app --since=1h
kubectl logs pod/my-app --since-time=2024-01-01T00:00:00Z

# 最近 N 行
kubectl logs --tail=100 pod/my-app

# 上一个容器（崩溃后查看）
kubectl logs --previous pod/my-app
```

## 成本监控（Kubecost）

```bash
helm install kubecost oci://ghcr.io/kubecost/charts/cost-analyzer \
  --namespace kubecost \
  --create-namespace
```

**Kubecost 提供：**
- 按命名空间/Deployment/Pod 的成本分配
- 集群资源利用率分析
- 成本优化建议（Rightsizing）
- 预算告警
