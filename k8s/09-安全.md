# Kubernetes 安全

## 安全体系概览

K8s 安全是多层次的，从外到内依次为：

```
传输层：TLS 加密（API Server 通信）
认证层：谁在访问（Authentication）
授权层：能做什么（Authorization）
准入层：能否通过（Admission Control）
运行层：容器安全（Pod Security / NetworkPolicy）
```

## RBAC（基于角色的访问控制）

### 核心对象

```
User / ServiceAccount ──绑定──> Role / ClusterRole ──包含──> 权限规则
```

| 资源 | 作用域 | 说明 |
|------|--------|------|
| **Role** | Namespace | 命名空间级别的权限集合 |
| **ClusterRole** | 集群 | 集群级别的权限集合 |
| **RoleBinding** | Namespace | 将 Role 绑定到 User/SA |
| **ClusterRoleBinding** | 集群 | 将 ClusterRole 绑定到 User/SA |

### Role 示例

```yaml
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  namespace: default
  name: pod-reader
rules:
  - apiGroups: [""]            # 核心 API 组
    resources: ["pods"]        # 资源类型
    verbs: ["get", "list", "watch"]  # 操作权限
  - apiGroups: ["apps"]
    resources: ["deployments"]
    verbs: ["get", "list", "watch"]
  - apiGroups: [""]
    resources: ["pods/log"]
    verbs: ["get", "list"]
```

### ClusterRole 示例

```yaml
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRole
metadata:
  name: cluster-admin-reader
rules:
  - apiGroups: [""]
    resources: ["nodes", "namespaces", "pods"]
    verbs: ["get", "list", "watch"]
  - apiGroups: ["metrics.k8s.io"]
    resources: ["pods", "nodes"]
    verbs: ["get", "list"]
```

### RoleBinding 示例

```yaml
apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata:
  name: read-pods
  namespace: default
subjects:
  - kind: User
    name: alice
    apiGroup: rbac.authorization.k8s.io
  - kind: ServiceAccount
    name: my-sa
    namespace: default
roleRef:
  kind: Role
  name: pod-reader
  apiGroup: rbac.authorization.k8s.io
```

### 常见内置 ClusterRole

| ClusterRole | 权限 |
|-------------|------|
| `cluster-admin` | 超级管理员（所有权限） |
| `admin` | 命名空间管理员 |
| `edit` | 可读写大多数资源 |
| `view` | 只读权限 |

```bash
# 给用户绑定只读权限
kubectl create clusterrolebinding alice-view \
  --clusterrole=view \
  --user=alice

# 给 ServiceAccount 绑定管理员权限
kubectl create rolebinding jenkins-admin \
  --clusterrole=admin \
  --serviceaccount=jenkins:default \
  --namespace=default
```

## ServiceAccount（服务账号）

ServiceAccount 是 Pod 中进程的身份标识：

```yaml
apiVersion: v1
kind: ServiceAccount
metadata:
  name: my-app-sa
  namespace: default
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: app
spec:
  template:
    spec:
      serviceAccountName: my-app-sa   # 指定 ServiceAccount
      automountServiceAccountToken: true  # 是否挂载 Token
      containers:
        - name: app
          image: my-app:latest
```

**默认行为：**
- 每个 Namespace 自动创建 `default` ServiceAccount
- Pod 未指定 SA 时使用 `default`
- Token 自动挂载到 `/var/run/secrets/kubernetes.io/serviceaccount/`

**安全建议：**
```yaml
# 对不需要访问 API 的 Pod，禁止挂载 Token
apiVersion: v1
kind: Pod
metadata:
  name: no-api-access
spec:
  automountServiceAccountToken: false
  containers:
    - name: worker
      image: busybox
```

## NetworkPolicy（网络策略）

NetworkPolicy 控制 Pod 之间的网络通信，**需要 CNI 插件支持**（Calico、Cilium、Weave 等）。

### 默认策略

```yaml
# 拒绝所有入站流量
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: default-deny-ingress
spec:
  podSelector: {}
  policyTypes:
    - Ingress
---
# 拒绝所有出站流量
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: default-deny-egress
spec:
  podSelector: {}
  policyTypes:
    - Egress
---
# 允许所有入站（宽松）
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-all-ingress
spec:
  podSelector: {}
  ingress:
    - {}
  policyTypes:
    - Ingress
```

### 常见隔离策略

```yaml
# 三层架构隔离
---
# 允许流量从 frontend 到 backend
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: backend-policy
  namespace: production
spec:
  podSelector:
    matchLabels:
      app: backend
  policyTypes:
    - Ingress
  ingress:
    - from:
        - podSelector:
            matchLabels:
              app: frontend
        - namespaceSelector:
            matchLabels:
              name: monitoring
      ports:
        - protocol: TCP
          port: 8080
---
# 允许流量从 backend 到 database
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: db-policy
  namespace: production
spec:
  podSelector:
    matchLabels:
      app: database
  policyTypes:
    - Ingress
  ingress:
    - from:
        - podSelector:
            matchLabels:
              app: backend
      ports:
        - protocol: TCP
          port: 3306
```

### 访问外部

```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-external-egress
spec:
  podSelector:
    matchLabels:
      app: backend
  policyTypes:
    - Egress
  egress:
    - to:
        - ipBlock:
            cidr: 10.0.0.0/8
            except:
              - 10.96.0.0/12    # 排除 ClusterIP 范围
    - to:
        - namespaceSelector: {}
```

## Pod Security Standards（Pod 安全标准）

K8s 内置三个安全等级：

| 等级 | 说明 |
|------|------|
| **Privileged** | 不受限制（系统组件、特权应用） |
| **Baseline** | 最小限制，禁止已知特权升级 |
| **Restricted** | 严格限制，符合 Pod 安全最佳实践 |

### Pod Security Admission

```yaml
# 命名空间级别强制策略
apiVersion: v1
kind: Namespace
metadata:
  name: restricted-ns
  labels:
    pod-security.kubernetes.io/enforce: restricted
    pod-security.kubernetes.io/audit: restricted
    pod-security.kubernetes.io/warn: baseline
```

```bash
# 查看命名空间的 Pod 安全级别
kubectl label namespace default \
  pod-security.kubernetes.io/enforce=restricted

# 会拒绝不满足 restricted 级别的 Pod
```

## Pod Security Context

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: secure-pod
spec:
  securityContext:
    runAsUser: 1000
    runAsGroup: 3000
    fsGroup: 2000
    seccompProfile:
      type: RuntimeDefault
    supplementalGroups: [4000]
  containers:
    - name: app
      image: nginx
      securityContext:
        runAsUser: 2000                # 覆盖 Pod 级别
        allowPrivilegeEscalation: false # 禁止特权提升
        privileged: false              # 非特权模式
        readOnlyRootFilesystem: true   # 只读根文件系统
        capabilities:
          drop: ["ALL"]                # 删除所有能力
          add: ["NET_BIND_SERVICE"]    # 只添加必要能力
        procMount: Default
```

## Kubernetes 安全清单

### 集群级别
- [ ] API Server 启用 TLS
- [ ] RBAC 启用（1.6+ 默认）
- [ ] 关闭匿名访问（`--anonymous-auth=false`）
- [ ] 限制 kubeconfig 分发
- [ ] etcd 启用认证和加密
- [ ] 使用私有镜像仓库

### 工作负载级别
- [ ] 不以 root 运行
- [ ] 只读根文件系统
- [ ] 删除多余 capabilities
- [ ] 设置资源 limits
- [ ] 使用 NetworkPolicy 隔离
- [ ] 启用 Pod Security Admission

### 运维级别
- [ ] 定期轮转证书
- [ ] 审计日志开启
- [ ] 镜像漏洞扫描
- [ ] 限制 Node 对外暴露端口
- [ ] 最小权限原则分配 RBAC

## 审计日志

```yaml
apiVersion: audit.k8s.io/v1
kind: Policy
metadata:
  name: audit-policy
rules:
  - level: RequestResponse
    resources:
      - group: ""
        resources: ["secrets"]
  - level: Metadata
    resources:
      - group: ""
        resources: ["pods", "configmaps"]
    omitStages:
      - RequestReceived
  - level: None
    users: ["system:kube-proxy"]
    verbs: ["watch"]
```

```bash
# 启用审计日志（kube-apiserver 参数）
--audit-policy-file=/etc/kubernetes/audit-policy.yaml
--audit-log-path=/var/log/kubernetes/audit.log
--audit-log-maxage=30
--audit-log-maxbackup=10
--audit-log-maxsize=100
```

## 生产安全建议

1. **最小权限原则**：每个 ServiceAccount 只给所需的权限
2. **默认拒绝策略**：先创建默认拒绝 NetworkPolicy，再开放必要策略
3. **镜像安全**：使用可信镜像，定期扫描漏洞（Trivy / Clair）
4. **运行时安全**：使用 Falco 检测异常行为
5. **密钥管理**：敏感数据用外部密钥管理（Vault / AWS Secrets Manager）
6. **网络隔离**：不同环境（dev/staging/prod）用不同 Namespace + NetworkPolicy
7. **证书管理**：使用 cert-manager 自动化证书管理
8. **API 限流**：配置 API Server 的 `--max-requests-inflight`
