# 配置管理

## ConfigMap 和 Secret 的作用

将配置从容器镜像中解耦，实现：
- 同一镜像在不同环境使用不同配置
- 敏感信息加密存储
- 配置热更新（无需重建镜像）

| 资源 | 用途 | 存储方式 | 大小限制 |
|------|------|----------|----------|
| **ConfigMap** | 非敏感配置 | 明文 | 1MB |
| **Secret** | 敏感信息（密码、密钥、证书） | Base64 / 加密 | 1MB |

## ConfigMap

### 创建 ConfigMap

```bash
# 从字面值创建
kubectl create configmap app-config \
  --from-literal=APP_ENV=production \
  --from-literal=LOG_LEVEL=info \
  --from-literal=DB_HOST=database.example.com

# 从文件创建
kubectl create configmap app-config \
  --from-file=application.properties \
  --from-file=config.yaml

# 从目录创建
kubectl create configmap app-config \
  --from-file=./config-dir/

# 从 .env 文件创建
kubectl create configmap app-config \
  --from-env-file=.env.production
```

### YAML 定义

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: app-config
  labels:
    app: my-app
data:
  # 键值对
  APP_ENV: production
  LOG_LEVEL: info
  DB_HOST: database.example.com
  DB_PORT: "3306"

  # 多行配置
  nginx.conf: |
    server {
      listen 80;
      server_name example.com;
      
      location / {
        proxy_pass http://backend:8080;
      }
    }

  # 配置文件键
  application.properties: |
    spring.datasource.url=jdbc:mysql://db:3306/myapp
    spring.datasource.maxActive=100
    server.port=8080
```

## Secret

### 创建 Secret

```bash
# 从字面值创建（会自动 Base64 编码）
kubectl create secret generic db-secret \
  --from-literal=username=admin \
  --from-literal=password='MyP@ssw0rd!'

# 从文件创建
kubectl create secret generic tls-secret \
  --from-file=tls.crt=server.crt \
  --from-file=tls.key=server.key

# 从 .dockerconfigjson 创建（镜像拉取密钥）
kubectl create secret docker-registry regcred \
  --docker-server=https://index.docker.io/v1/ \
  --docker-username=myuser \
  --docker-password=mypassword \
  --docker-email=user@example.com

# TLS 证书
kubectl create secret tls tls-secret \
  --cert=path/to/tls.crt \
  --key=path/to/tls.key
```

### YAML 定义

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: db-secret
type: Opaque                       # 默认类型
data:
  username: YWRtaW4=               # admin 的 Base64
  password: TXlQQHNzdzByZCE=       # MyP@ssw0rd! 的 Base64

---
# 使用 stringData 可以写明文（自动编码）
apiVersion: v1
kind: Secret
metadata:
  name: db-secret
type: Opaque
stringData:
  username: admin
  password: MyP@ssw0rd!
```

### Secret 类型

| 类型 | 用途 |
|------|------|
| `Opaque` | 任意键值对 |
| `kubernetes.io/dockerconfigjson` | Docker 镜像仓库认证 |
| `kubernetes.io/tls` | TLS 证书 |
| `kubernetes.io/ssh-auth` | SSH 认证 |
| `kubernetes.io/basic-auth` | HTTP Basic Auth |
| `bootstrap.kubernetes.io/token` | Bootstrap Token |

## 使用配置的方式

### 方式一：环境变量

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: config-demo
spec:
  containers:
    - name: app
      image: my-app:latest
      env:
        # 从 ConfigMap 读取单个值
        - name: APP_ENV
          valueFrom:
            configMapKeyRef:
              name: app-config
              key: APP_ENV
        # 从 Secret 读取单个值
        - name: DB_PASSWORD
          valueFrom:
            secretKeyRef:
              name: db-secret
              key: password
      # 批量导入所有键值对
      envFrom:
        - configMapRef:
            name: app-config
        - secretRef:
            name: app-secret
            optional: true     # Secret 不存在时 Pod 仍可启动
```

### 方式二：挂载为文件

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: config-volume-demo
spec:
  containers:
    - name: app
      image: my-app:latest
      volumeMounts:
        - name: config-volume
          mountPath: /etc/config
          readOnly: true
        - name: secret-volume
          mountPath: /etc/secret
          readOnly: true
        - name: nginx-config
          mountPath: /etc/nginx/conf.d
          readOnly: true
  volumes:
    - name: config-volume
      configMap:
        name: app-config
    - name: secret-volume
      secret:
        secretName: db-secret
        defaultMode: 0400       # 文件权限（8 进制）
        items:                  # 只挂载特定键
          - key: password
            path: db-password
            mode: 0400
    - name: nginx-config
      configMap:
        name: nginx-config
        items:
          - key: nginx.conf
            path: default.conf
```

### 方式三：命令行参数

```yaml
spec:
  containers:
    - name: app
      image: my-app:latest
      command:
        - /app/start.sh
      args:
        - "--db-host=$(DB_HOST)"
        - "--db-port=$(DB_PORT)"
      env:
        - name: DB_HOST
          valueFrom:
            configMapKeyRef:
              name: app-config
              key: DB_HOST
        - name: DB_PORT
          valueFrom:
            configMapKeyRef:
              name: app-config
              key: DB_PORT
```

## ConfigMap/Secret 热更新

**挂载为文件时支持热更新**（环境变量方式不支持）：

```bash
# ConfigMap 更新后，挂载的文件会在几分钟内自动更新
kubectl edit configmap app-config

# 但进程需要自己检测文件变更，或者由 Sidecar 触发 reload
# Nginx 示例：
kubectl exec -it nginx-pod -- nginx -s reload
```

**触发滚动更新（推荐方式）：**
```yaml
# 使用 annotation 变更触发 Pod 重建
spec:
  template:
    metadata:
      annotations:
        checksum/config: ${CONFIG_CHECKSUM}
```

## 镜像拉取密钥（ImagePullSecret）

```bash
# 创建 Secret
kubectl create secret docker-registry regcred \
  --docker-server=https://registry.cn-hangzhou.aliyuncs.com \
  --docker-username=myuser \
  --docker-password=mypassword

# 方式一：在 Pod 中引用
spec:
  imagePullSecrets:
    - name: regcred
  containers:
    - name: app
      image: registry.cn-hangzhou.aliyuncs.com/myapp:v1

# 方式二：添加到 ServiceAccount（默认自动注入）
kubectl patch serviceaccount default \
  -p '{"imagePullSecrets": [{"name": "regcred"}]}'
```

## 生产建议

1. **ConfigMap 大小限制 1MB**，大配置用 Volume 挂载文件
2. **Secret 不要提交到代码仓库**，使用外部密钥管理（Vault / AWS Secrets Manager）
3. **Secret 默认非加密**，etcd 中的 Secret 只是 Base64 编码，应开启 etcd 加密
4. **环境差异通过不同的 ConfigMap/Secret 区分**，不要修改 YAML 内容
5. **使用 KMS 加密 Secret**（云厂商通常提供）
6. **配置变更尽量触发滚动更新**，而不是依赖热更新
7. **不要用 Secret 存大量数据**，1MB 限制是硬限制

### 开启 etcd 加密

```yaml
# kube-apiserver 配置
apiVersion: apiserver.config.k8s.io/v1
kind: EncryptionConfiguration
resources:
  - resources:
      - secrets
    providers:
      - aescbc:
          keys:
            - name: key1
              secret: <base64-encoded-key>
      - identity: {}  # 回退到明文
```
