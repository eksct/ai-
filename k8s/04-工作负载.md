# 工作负载（Workloads）

## 总览

K8s 提供多种工作负载控制器来管理 Pod 的生命周期：

| 控制器 | 适用场景 | 特点 |
|--------|----------|------|
| **Deployment** | 无状态应用 | 滚动更新、回滚、扩缩容 |
| **StatefulSet** | 有状态应用 | 稳定网络标识、有序部署 |
| **DaemonSet** | 每个节点运行一个 | 日志采集、监控代理 |
| **Job** | 一次性任务 | 运行到完成 |
| **CronJob** | 定时任务 | 基于 Cron 表达式 |

---

## Deployment（无状态应用）

最常用的工作负载，适合 Web 服务、API 后端等无状态应用。

### 基本示例

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
  labels:
    app: nginx
spec:
  replicas: 3                    # 期望副本数
  selector:
    matchLabels:
      app: nginx                # 匹配 Pod 标签
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
        - name: nginx
          image: nginx:1.25-alpine
          ports:
            - containerPort: 80
          resources:
            requests:
              cpu: 100m
              memory: 128Mi
            limits:
              cpu: 500m
              memory: 256Mi
```

### 滚动更新

```yaml
spec:
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxUnavailable: 1         # 更新时最多不可用 1 个
      maxSurge: 1               # 更新时最多超出 1 个
  minReadySeconds: 30           # 新 Pod 就绪后等待 30s 才算可用
  revisionHistoryLimit: 10      # 保留历史版本数
```

```bash
# 触发滚动更新
kubectl set image deployment/nginx-deployment nginx=nginx:1.26-alpine
# 或更新 YAML 后 apply
kubectl apply -f deployment.yaml

# 查看更新状态
kubectl rollout status deployment/nginx-deployment

# 查看历史版本
kubectl rollout history deployment/nginx-deployment

# 回滚到上一版本
kubectl rollout undo deployment/nginx-deployment

# 回滚到指定版本
kubectl rollout undo deployment/nginx-deployment --to-revision=2

# 暂停/恢复更新
kubectl rollout pause deployment/nginx-deployment
kubectl rollout resume deployment/nginx-deployment
```

### 扩缩容

```bash
# 手动扩容
kubectl scale deployment/nginx-deployment --replicas=5

# 自动扩缩容（HPA）
kubectl autoscale deployment/nginx-deployment \
  --cpu-percent=80 \
  --min=3 \
  --max=10
```

### HPA（Horizontal Pod Autoscaler）

```yaml
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: nginx-hpa
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: nginx-deployment
  minReplicas: 3
  maxReplicas: 20
  metrics:
    - type: Resource
      resource:
        name: cpu
        target:
          type: Utilization
          averageUtilization: 70
    - type: Resource
      resource:
        name: memory
        target:
          type: AverageValue
          averageValue: 512Mi
    - type: Pods
      pods:
        metric:
          name: requests-per-second
        target:
          type: AverageValue
          averageValue: 1000
```

## StatefulSet（有状态应用）

适合数据库、ZooKeeper、Kafka 等需要稳定身份和存储的应用。

```yaml
apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: mysql
spec:
  serviceName: mysql-h        # Headless Service 名称
  replicas: 3
  selector:
    matchLabels:
      app: mysql
  template:
    metadata:
      labels:
        app: mysql
    spec:
      containers:
        - name: mysql
          image: mysql:8.0
          ports:
            - containerPort: 3306
              name: mysql
          env:
            - name: MYSQL_ROOT_PASSWORD
              valueFrom:
                secretKeyRef:
                  name: mysql-secret
                  key: root-password
          volumeMounts:
            - name: data
              mountPath: /var/lib/mysql
  volumeClaimTemplates:       # 自动创建 PVC
    - metadata:
        name: data
      spec:
        accessModes: ["ReadWriteOnce"]
        storageClassName: standard
        resources:
          requests:
            storage: 10Gi
```

### StatefulSet 的特点

| 特性 | 说明 |
|------|------|
| **Pod 名称** | `mysql-0`, `mysql-1`, `mysql-2`，稳定有序 |
| **DNS 名称** | `mysql-0.mysql-h.default.svc.cluster.local` |
| **启动顺序** | 从 0 到 N-1 顺序创建 |
| **停止顺序** | 从 N-1 到 0 逆序终止 |
| **存储** | 每个 Pod 绑定独立的 PVC |
| **删除保护** | 不会自动删除 PVC（需手动清理） |

### Headless Service

```yaml
apiVersion: v1
kind: Service
metadata:
  name: mysql-h
spec:
  clusterIP: None              # Headless：不做负载均衡
  selector:
    app: mysql
  ports:
    - port: 3306
```

## DaemonSet（每个节点一个 Pod）

适合日志采集（Fluentd）、监控（Prometheus Node Exporter）、网络插件（Calico）等。

```yaml
apiVersion: apps/v1
kind: DaemonSet
metadata:
  name: fluentd
  namespace: kube-system
spec:
  selector:
    matchLabels:
      name: fluentd
  template:
    metadata:
      labels:
        name: fluentd
    spec:
      tolerations:
        - operator: Exists     # 容忍所有污点，在所有节点运行
      serviceAccountName: fluentd
      containers:
        - name: fluentd
          image: fluent/fluentd:v1.16
          resources:
            limits:
              memory: 512Mi
              cpu: 500m
            requests:
              memory: 256Mi
              cpu: 200m
          volumeMounts:
            - name: varlog
              mountPath: /var/log
            - name: containers
              mountPath: /var/log/containers
      terminationGracePeriodSeconds: 30
      volumes:
        - name: varlog
          hostPath:
            path: /var/log
        - name: containers
          hostPath:
            path: /var/log/containers
```

**常用场景：**
- 日志采集（Fluentd / Filebeat / Logstash）
- 监控采集（Prometheus Node Exporter / Datadog Agent）
- 网络插件（Calico / Flannel / Cilium）
- 安全代理（Aqua / Twistlock）
- CNI 插件（kube-proxy 本身也是 DaemonSet）

## Job（一次性任务）

```yaml
apiVersion: batch/v1
kind: Job
metadata:
  name: pi-job
spec:
  completions: 1              # 需要成功完成几次
  parallelism: 1              # 并行执行几个 Pod
  backoffLimit: 4             # 失败重试次数
  activeDeadlineSeconds: 100  # 任务超时时间
  template:
    spec:
      containers:
        - name: pi
          image: perl:5.34
          command: ["perl", "-Mbignum=bpi", "-wle", "print bpi(2000)"]
      restartPolicy: Never     # Job 只能是 Never 或 OnFailure
```

**并行 Job 示例：**
```yaml
apiVersion: batch/v1
kind: Job
metadata:
  name: parallel-job
spec:
  completions: 10             # 总共需要完成 10 次
  parallelism: 3              # 同时运行 3 个 Pod
  template:
    spec:
      containers:
        - name: worker
          image: busybox
          command: ["sh", "-c", "echo Job $JOB_COMPLETION_INDEX && sleep 5"]
      restartPolicy: Never
```

## CronJob（定时任务）

```yaml
apiVersion: batch/v1
kind: CronJob
metadata:
  name: backup-job
spec:
  schedule: "0 3 * * *"       # 每天凌晨 3 点执行
  timeZone: "Asia/Shanghai"   # 指定时区
  startingDeadlineSeconds: 100 # 启动截止时间
  concurrencyPolicy: Forbid   # 禁止并发（上次未完成则跳过）
  successfulJobsHistoryLimit: 3
  failedJobsHistoryLimit: 1
  suspend: false              # 是否暂停
  jobTemplate:
    spec:
      template:
        spec:
          containers:
            - name: backup
              image: busybox
              command: ["sh", "-c", "echo Running backup..."]
          restartPolicy: OnFailure
```

**concurrencyPolicy 说明：**
- `Allow`（默认）：允许并发执行
- `Forbid`：禁止并发，上次未完成时跳过
- `Replace`：替换上次还在运行的 Job

## 控制器选择指南

```
应用类型
  ├── 无状态（Web API, 微服务）     → Deployment + HPA
  ├── 有状态（MySQL, Redis, Kafka） → StatefulSet + Headless Service
  ├── 守护进程（日志/监控 Agent）    → DaemonSet
  ├── 批处理任务（数据处理）         → Job
  └── 定时任务（备份, 清理, 报表）   → CronJob
```
