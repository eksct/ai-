# 异常处理

## 异常体系

```
Throwable
 ├── Error（严重问题，程序无法处理）
 │    ├── OutOfMemoryError
 │    ├── StackOverflowError
 │    └── NoClassDefFoundError
 └── Exception（程序可处理）
      ├── RuntimeException（非受检异常）
      │    ├── NullPointerException
      │    ├── ArrayIndexOutOfBoundsException
      │    ├── IllegalArgumentException
      │    ├── ClassCastException
      │    ├── NumberFormatException
      │    └── ArithmeticException
      └── 受检异常（Checked Exception）
           ├── IOException
           ├── SQLException
           ├── FileNotFoundException
           └── ClassNotFoundException
```

## 异常分类

| 类型 | 说明 | 处理要求 |
|------|------|----------|
| **Error** | JVM 内部错误，程序无法恢复 | 不处理 |
| **非受检异常** (RuntimeException) | 代码逻辑问题，可避免 | 编译器不强制处理 |
| **受检异常** (Checked Exception) | 外部因素导致，不可控 | 编译器强制处理（throws / try-catch） |

## 异常处理方式

### try-catch-finally

```java
try {
    // 可能抛出异常的代码
    int result = 10 / 0;
} catch (ArithmeticException e) {
    // 捕获特定异常
    System.err.println("除零异常: " + e.getMessage());
} catch (NullPointerException | IllegalArgumentException e) {
    // 多异常捕获（Java 7+）
    System.err.println("异常: " + e);
} catch (Exception e) {
    // 捕获通用异常（放在最后）
    e.printStackTrace();
} finally {
    // 无论是否异常都会执行（释放资源）
    System.out.println("总是执行");
}
```

### try-with-resources（Java 7+）

自动关闭实现了 `AutoCloseable` 的资源：

```java
// 传统方式（手动 close）
BufferedReader br = null;
try {
    br = new BufferedReader(new FileReader("test.txt"));
    System.out.println(br.readLine());
} catch (IOException e) {
    e.printStackTrace();
} finally {
    if (br != null) {
        try { br.close(); } catch (IOException e) {}
    }
}

// try-with-resources（自动关闭）
try (BufferedReader br = new BufferedReader(new FileReader("test.txt"))) {
    System.out.println(br.readLine());
} catch (IOException e) {
    e.printStackTrace();
}
// br 自动关闭
```

多个资源：
```java
try (FileInputStream fis = new FileInputStream("a.txt");
     FileOutputStream fos = new FileOutputStream("b.txt")) {
    byte[] buf = new byte[1024];
    int len;
    while ((len = fis.read(buf)) != -1) {
        fos.write(buf, 0, len);
    }
}
```

### throws / throw

```java
// throws：声明方法可能抛出的异常
public void readFile(String path) throws FileNotFoundException, IOException {
    if (path == null) {
        throw new IllegalArgumentException("路径不能为空");  // throw：抛出异常
    }
    Files.readAllLines(Paths.get(path));
}

// 调用者必须处理
public void callReadFile() {
    try {
        readFile("test.txt");
    } catch (IOException e) {
        e.printStackTrace();
    }
}
```

## 自定义异常

```java
// 自定义受检异常
public class BusinessException extends Exception {
    private int errorCode;

    public BusinessException(int errorCode, String message) {
        super(message);
        this.errorCode = errorCode;
    }

    public int getErrorCode() {
        return errorCode;
    }
}

// 自定义非受检异常
public class ValidationException extends RuntimeException {
    public ValidationException(String message) {
        super(message);
    }
}

// 使用
public void transferMoney(double amount) throws BusinessException {
    if (amount <= 0) {
        throw new ValidationException("金额必须大于零");
    }
    if (amount > balance) {
        throw new BusinessException(1001, "余额不足");
    }
}
```

## 异常链

```java
try {
    // 底层异常
    throw new SQLException("数据库连接失败");
} catch (SQLException e) {
    // 包装为高层异常，保留原始异常信息
    throw new BusinessException(5000, "操作失败", e);
}
```

## 异常最佳实践

### 1. 不要捕获 Exception 或 Throwable
```java
// ❌ 不推荐
try {
    // ...
} catch (Exception e) {  // 太宽泛
    // ...
}

// ✅ 推荐
try {
    // ...
} catch (IOException | SQLException e) {  // 精确捕获
    // ...
}
```

### 2. 不要忽略异常
```java
// ❌ 不推荐（空 catch）
try {
    // ...
} catch (Exception e) {}

// ✅ 至少记录日志
try {
    // ...
} catch (Exception e) {
    log.error("操作失败", e);
    throw e;  // 或者抛出封装后的异常
}
```

### 3. 不要用异常控制流程
```java
// ❌ 不推荐
try {
    int index = Integer.parseInt(userInput);
    // ...
} catch (NumberFormatException e) {
    // 用异常判断输入是否是数字
}

// ✅ 推荐
if (userInput != null && userInput.matches("\\d+")) {
    int index = Integer.parseInt(userInput);
    // ...
}
```

### 4. 尽早抛出，稍晚捕获
```java
public void process(String input) {
    // 参数验证尽早抛出
    if (input == null || input.isBlank()) {
        throw new IllegalArgumentException("input 不能为空");
    }
    // ...
}
```

### 5. 使用 try-with-resources
```java
// ✅ 始终使用 try-with-resources 代替 finally 关闭资源
try (Connection conn = getConnection();
     PreparedStatement stmt = conn.prepareStatement(sql)) {
    // ...
}
```

## 常见异常及原因

| 异常 | 常见原因 |
|------|----------|
| `NullPointerException` | 调用 null 对象的方法或属性 |
| `ArrayIndexOutOfBoundsException` | 访问数组越界索引 |
| `StringIndexOutOfBoundsException` | 访问字符串越界索引 |
| `NumberFormatException` | 字符串转数字格式错误 |
| `ClassCastException` | 类型转换不兼容 |
| `IllegalArgumentException` | 方法参数不合法 |
| `ArithmeticException` | 除零等算术异常 |
| `ConcurrentModificationException` | 遍历时修改集合 |
| `StackOverflowError` | 递归调用过深 |
| `OutOfMemoryError` | 堆内存不足 |
