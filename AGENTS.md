# 工作指南

## 通用原则

你是这个项目的笔记助手。用户让你创建或修改笔记时，必须遵循以下流程。

## 创建/修改笔记的自动流程

当用户要求创建或修改 `docker/`、`k8s/`、`生活常识/`、`golang/` 目录下的笔记时，在**所有文件操作完成后**，自动执行以下步骤（**不需要用户再次提醒**）：

### 步骤 1：审阅

检查本次新增/变更的 `.md` 文件：

- 是否包含一级标题（`# 标题`）
- 内容是否过短（少于 200 字应提醒）
- 明显的格式问题

如果有问题，向用户报告并询问是否修正。

### 步骤 2：确认

向用户报告变更摘要：
- 新增了哪些文件
- 修改了哪些文件
- 审阅结果

询问"是否提交并推送到 Git 仓库？"

### 步骤 3：提交并推送

用户确认后，执行：

```bash
git add -A
git commit -m "add: 新增 xxx 笔记"   # 根据实际变更生成有意义的提交信息
git push
```

如果推送失败，向用户报告错误原因。

## 不需要自动推送的场景

以下情况**不要**自动执行推送：
- 用户只是询问问题、搜索内容
- 用户明确说"不要推送"或"先不提交"
- 文件变更不属于上述笔记目录
