# Docker 最佳实践与安全

## 一、Dockerfile 最佳实践

### 1. 选择合适的基础镜像

```dockerfile
# ❌ 不推荐：大而全
FROM ubuntu:22.04

# ✅ 推荐：小而精
FROM alpine:latest
FROM node:18-alpine
FROM python:3.11-slim
FROM golang:1.21-alpine
```

- Alpine 约 5MB，Ubuntu 约 70MB
- 选择官方镜像，确保安全性
- 固定标签版本，不要使用 `latest`

### 2. 减少镜像层数

```dockerfile
# ❌ 不推荐：多层
RUN apt-get update
RUN apt-get install -y package1
RUN apt-get install -y package2
RUN rm -rf /var/lib/apt/lists/*

# ✅ 推荐：合并
RUN apt-get update && \
    apt-get install -y package1 package2 && \
    rm -rf /var/lib/apt/lists/*
```

### 3. 合理利用构建缓存

```dockerfile
# ❌ 不推荐：源码变动导致 npm install 缓存失效
COPY . .
RUN npm install

# ✅ 推荐：先复制依赖文件
COPY package.json package-lock.json ./
RUN npm install
COPY . .
```

### 4. 多阶段构建

```dockerfile
# 构建阶段
FROM golang:1.21 AS builder
WORKDIR /app
COPY . .
RUN CGO_ENABLED=0 go build -o app

# 运行阶段
FROM alpine:latest
RUN apk --no-cache add ca-certificates
COPY --from=builder /app/app .
USER nobody
CMD ["./app"]
```

### 5. 非 root 用户运行

```dockerfile
# ❌ 不推荐
FROM node:18
COPY . .
RUN npm install
CMD ["node", "app.js"]

# ✅ 推荐
FROM node:18
RUN groupadd -r app && useradd -r -g app app
COPY . .
RUN npm install && chown -R app:app /app
USER app
CMD ["node", "app.js"]
```

### 6. 使用 .dockerignore

```

.gitignore
Dockerfile
.dockerignore
.env
dist
.cache
```

## 二、运行时最佳实践

### 1. 资源限制

```bash
# 始终设置资源限制
docker run -d \
  --memory="512m" \
  --memory-reservation="256m" \
  --cpus="0.5" \
  --pids-limit=100 \
  --restart=unless-stopped \
  nginx
```

### 2. 只读根文件系统

```bash
docker run -d \
  --read-only \
  --tmpfs /tmp:rw,noexec,nosuid,size=64m \
  --tmpfs /var/run:rw,noexec,nosuid,size=64m \
  nginx
```

### 3. 安全配置

```bash
# 移除所有能力
docker run -d --cap-drop ALL nginx

# 仅添加必要能力
docker run -d --cap-drop ALL --cap-add NET_BIND_SERVICE nginx

# 使用安全选项
docker run -d \
  --security-opt no-new-privileges:true \
  --security-opt seccomp=default.json \
  nginx
```

### 4. 健康检查

```dockerfile
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
  CMD curl -f http://localhost:8080/health || exit 1
```

## 三、安全管理

### 1. 镜像安全

```bash
# 扫描镜像漏洞
docker scan nginx:latest

# 使用 Trivy 扫描
trivy image nginx:latest

# 只在可信仓库拉取
docker pull nginx:alpine
```

### 2. 密钥管理

```bash
# ❌ 不推荐：环境变量传递密码
docker run -e MYSQL_ROOT_PASSWORD=mysecret mysql

# ✅ 推荐：使用 Docker Secrets（Swarm）
echo "mysecret" | docker secret create mysql_password -

# ✅ 推荐：使用文件挂载
docker run -v /run/secrets:/run/secrets:ro mysql
```

### 3. Docker 守护进程安全

```bash
# 避免暴露 Docker API
# 不设置 -H tcp://0.0.0.0:2375

# 如果必须远程访问，启用 TLS
dockerd \
  --tlsverify \
  --tlscacert=ca.pem \
  --tlscert=server-cert.pem \
  --tlskey=server-key.pem \
  -H=0.0.0.0:2376
```

### 4. rootless 模式

```bash
# 安装 rootless 模式
dockerd-rootless-setuptool.sh install

# 以非 root 用户运行 Docker
systemctl --user start docker
```

## 四、运维最佳实践

### 1. 日志管理

```bash
# 限制日志大小
docker run -d \
  --log-driver json-file \
  --log-opt max-size=10m \
  --log-opt max-file=3 \
  nginx

# 生产环境使用集中式日志
docker run -d \
  --log-driver fluentd \
  --log-opt fluentd-address=localhost:24224 \
  nginx
```

### 2. 监控

```bash
# 查看容器资源使用
docker stats

# 使用 cAdvisor 监控所有容器
docker run -d \
  --name cadvisor \
  -p 8080:8080 \
  -v /:/rootfs:ro \
  -v /var/run:/var/run:ro \
  -v /sys:/sys:ro \
  -v /var/lib/docker:/var/lib/docker:ro \
  gcr.io/cadvisor/cadvisor
```

### 3. 清理

```bash
# 定期清理未使用的资源
docker system prune -a --volumes

# 可以设置定时任务
# Linux cron: 0 3 * * 0 /usr/bin/docker system prune -f
```

### 4. 备份

```bash
# 备份卷数据
docker run --rm \
  -v my-volume:/data \
  -v /backup:/backup \
  ubuntu tar czf /backup/volume-backup-$(date +%Y%m%d).tar.gz /data
```

## 五、性能优化

### 1. 镜像优化

```bash
# 使用 --squash 合并层（实验性）
docker build --squash -t my-app .

# 使用 BuildKit
DOCKER_BUILDKIT=1 docker build -t my-app .
```

### 2. 网络优化

```bash
# 使用 host 网络（性能要求高的场景）
docker run --network host nginx

# 自定义 bridge 优于默认 bridge
docker network create --driver bridge my-network
```

### 3. 存储驱动

```bash
# 确保使用 overlay2
docker info | grep "Storage Driver"
# 输出应为 Storage Driver: overlay2
```

## 六、常见陷阱

### 1. 不要在容器中存储数据
```bash
# ❌ 不推荐
docker run -d mysql

# ✅ 推荐
docker run -d -v mysql-data:/var/lib/mysql mysql
```

### 2. 不要以 root 运行
```bash
# ❌ 不推荐
docker run -d nginx

# ✅ 推荐 - 确保 Dockerfile 中有 USER 指令
```

### 3. 不要使用 :latest 标签
```bash
# ❌ 不推荐
FROM node:latest

# ✅ 推荐
FROM node:18.17.0-alpine
```

### 4. 不要忽略健康检查
```bash
# ❌ 不推荐 - 不设置健康检查
docker run -d nginx

# ✅ 推荐
docker run -d --health-cmd="curl -f http://localhost/health" nginx
```
