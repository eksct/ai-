# 什么是 Docker

## 概述

Docker 是一个开源的容器化平台，用于开发、打包、部署和运行应用程序。它通过容器技术将应用及其依赖打包在一起，确保应用在任何环境中都能一致地运行。

## 核心概念

### 1. 容器 (Container)
- 轻量级的可执行软件包，包含运行应用所需的一切：代码、运行时、系统工具、库和设置
- 与虚拟机不同，容器共享宿主机的 OS 内核，无需完整操作系统
- 启动快（毫秒级），资源占用少

### 2. 镜像 (Image)
- 容器的只读模板，用于创建容器
- 包含应用程序及其依赖的完整文件系统
- 通过 Dockerfile 定义和构建
- 分层存储，复用已有层

### 3. 仓库 (Registry)
- 存储和分发 Docker 镜像的地方
- Docker Hub 是官方公共仓库
- 可搭建私有仓库 (Harbor, Nexus 等)

## Docker vs 虚拟机

| 特性 | Docker 容器 | 虚拟机 |
|------|------------|--------|
| 启动速度 | 毫秒级 | 分钟级 |
| 磁盘占用 | MB 级 | GB 级 |
| 性能 | 近乎原生 | 有一定损耗 |
| 内核 | 共享宿主机内核 | 独立内核 |
| 隔离性 | 进程级隔离 | 完全隔离 |

## Docker 架构

```
+-------------------+       +-------------------+
|   Docker Client   | ----> |   Docker Daemon   |
| (docker CLI)      |       | (dockerd)         |
+-------------------+       +-------------------+
                                    |
                    +---------------+---------------+
                    |               |               |
              +-----------+   +-----------+   +-----------+
              |  Images   |   | Containers|   |  Volumes  |
              +-----------+   +-----------+   +-----------+
                    |               |
              +-----------+         |
              | Registry  |<--------+
              +-----------+
```

- **Docker Daemon (dockerd)**: 后台守护进程，管理镜像、容器、网络、存储等
- **Docker Client (docker)**: 用户与 Docker 交互的命令行工具
- **REST API**: Client 与 Daemon 通过 REST API 通信
- **Registry**: 镜像仓库，存储和分发镜像

## 为什么使用 Docker

1. **环境一致性**：消除"在我机器上能跑"的问题
2. **快速部署**：秒级启动容器
3. **资源高效**：比虚拟机更轻量，更高密度
4. **易于扩展**：结合编排工具轻松扩缩容
5. **版本控制**：镜像可版本化管理
6. **微服务友好**：每个服务独立容器化
