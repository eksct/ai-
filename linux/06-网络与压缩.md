# Linux 网络与压缩

## 概述

网络命令用来查连接、测联通、抓数据。压缩命令用来打包分发文件。两个都是运维高频操作。

## 网络命令

### ping — 测试网络连通性

```bash
ping 8.8.8.8                      # 持续 ping，Ctrl+C 退出
ping -c 4 8.8.8.8                 # 只 ping 4 次
ping -c 4 baidu.com               # 也测 DNS 解析
```

### curl — HTTP 请求工具

```bash
# 最常用
curl -I https://api.example.com           # 只查响应头（看状态码）
curl -v https://api.example.com           # 详细输出（调试用）
curl -o output.html https://example.com   # 下载到文件
curl -X POST -d '{"key":"value"}' -H "Content-Type: application/json" https://api.example.com/create
# ↑ 发 POST 请求

# 排查问题
curl -w "HTTP %{http_code} %{time_total}s\n" https://api.example.com
# ↑ 显示状态码和总耗时

curl --connect-timeout 5 --max-time 10 https://slow-api.com
# ↑ 限制连接超时 5 秒，总超时 10 秒
```

### wget — 下载文件

```bash
wget -O myfile.zip https://example.com/file.zip    # 指定输出文件名
wget -c https://example.com/bigfile.iso             # 断点续传
wget -r -np https://example.com/files/              # 递归下载目录
```

`curl` 和 `wget` 选哪个？日常随手测试用 `curl`，下载大文件用 `wget`（支持断点续传）。

### ssh — 远程连接

```bash
ssh user@192.168.1.10                  # 基本登录
ssh -i ~/.ssh/id_rsa user@host         # 指定密钥
ssh -p 2222 user@host                  # 指定端口
ssh -J jump-host user@target-host      # 跳板机登录（堡垒机场景）
```

### scp — 远程复制文件

```bash
scp file.txt user@host:/data/          # 本地上传到远程
scp user@host:/data/file.txt ./        # 远程下载到本地
scp -r /data/app user@host:/data/      # 递归复制目录
scp -P 2222 file.txt user@host:/data/  # 指定端口
```

### rsync — 增量同步（推荐替代 scp）

```bash
rsync -avz /data/ user@host:/backup/   # 增量同步，只传差异部分
rsync -avz --delete /data/ user@host:/backup/
# ↑ 远程有而本地没有的，删掉（保持精确镜像）
```

`rsync` 比 `scp` 强的地方：只传变化的部分、支持压缩、失败可续传。大数据量传输用 `rsync`。

### netstat / ss — 查看网络连接

```bash
netstat -tlnp                         # 看哪些端口在监听（常用）
# -t TCP, -l 监听, -n 数字显示, -p 显示进程

ss -tlnp                              # netstat 的现代替代（更快）

# 场景：查 8080 端口被谁占了
sudo lsof -i :8080                    # 显示占用 8080 的进程
# COMMAND   PID   USER   FD   TYPE  DEVICE SIZE/OFF NODE NAME
# java    12345 deploy  123u  IPv4  12345      0t0  TCP *:8080 (LISTEN)
```

### 其他网络工具

```bash
hostname                              # 本机主机名
hostname -I                           # 本机 IP

dig example.com                       # DNS 查询（比 nslookup 详细）
dig -x 8.8.8.8                        # 反向 DNS 查询

# 查路由路径
traceroute 8.8.8.8                    # 追踪到目标经过哪些路由器
mtr 8.8.8.8                           # traceroute + ping 合体

# 端口连通性测试
telnet 192.168.1.10 6379              # 能连上说明端口通
nc -zv 192.168.1.10 6379              # 更现代的方式
```

## 压缩与打包

### tar — 打包（不压缩）

```bash
tar cf archive.tar /path/to/dir       # 打包（c=create, f=file）
tar tf archive.tar                    # 查看包内容（t=list）
tar xf archive.tar                    # 解包（x=extract）
```

### 打包 + 压缩（最常用）

```bash
# 打包并压缩
tar czf archive.tar.gz /path/to/dir     # gzip 压缩（最快的，推荐）
tar cjf archive.tar.bz2 /path/to/dir    # bzip2 压缩（更小但慢）
tar cJf archive.tar.xz /path/to/dir     # xz 压缩（最小但最慢）

# 解压
tar xzf archive.tar.gz                  # 解压 gzip
tar xjf archive.tar.bz2                 # 解压 bzip2
tar xJf archive.tar.xz                  # 解压 xz

# 常用组合
tar czf app-$(date +%Y%m%d).tar.gz /data/app    # 备份 app 目录，文件名带日期
tar xzf app-20260624.tar.gz -C /tmp/restore     # 解压到指定目录
```

### gzip / gunzip — 压缩单个文件

```bash
gzip access.log                       # 压缩成 access.log.gz（原文件消失）
gunzip access.log.gz                  # 解压

# 压缩级别 1-9（1 最快，9 最小，默认 6）
gzip -1 fast.log                      # 快速压缩
gzip -9 small.log                     # 最小压缩

# 查看压缩文件内容
zcat access.log.gz | grep "ERROR"     # 不解压直接搜
zless access.log.gz                   # 分页查看
```

### zip — Windows 兼容压缩

```bash
zip -r archive.zip /path/to/dir       # 递归压缩
unzip archive.zip                     # 解压到当前目录
unzip archive.zip -d /target/dir      # 解压到指定目录
```

## 生产实践

### 常用组合

```bash
# 排查端口冲突
sudo lsof -i :8080
sudo ss -tlnp | grep 8080

# 远程同步代码
rsync -avz --exclude 'node_modules' --exclude '.git' ./ user@host:/app/

# 备份并压缩
tar czf backup-$(date +%Y%m%d-%H%M%S).tar.gz /data/app /data/db

# 传输大文件时实时看进度
rsync -avz --progress bigfile.tar.gz user@host:/data/

# 看某个端口谁在连
ss -tunp | grep :6379
```

### 踩坑记录

1. **`scp` 传大文件断了重来**：`scp` 不支持断点续传，传了一半断了就得从头开始。大文件用 `rsync` 或 `wget -c`
2. **`tar` 解压覆盖已有文件**：`tar xzf` 默认直接覆盖，没有确认。先用 `tar tf` 查看包内容，确认路径正确再解压
3. **`curl` 超时不设**：默认 `curl` 没有超时，可能卡住几分钟。生产脚本里永远加 `--connect-timeout` 和 `--max-time`
4. **SSH 密钥权限**：`id_rsa` 权限不对（不是 600），SSH 直接拒绝使用。`chmod 600 ~/.ssh/id_rsa`
5. **压缩工具选错**：`gzip` 只能压单个文件，目录需要用 `tar` 先打包再压缩。每次用 `tar czf` 即可

## 总结

网络排查用 `curl` + `ss -tlnp` + `lsof -i :port`。文件传输大数据用 `rsync`，小文件用 `scp`。压缩用 `tar czf` 这一个组合就够了，不用记别的参数。
