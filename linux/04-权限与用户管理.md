# Linux 权限与用户管理

## 概述

Linux 是多用户系统，权限控制决定谁能读/写/执行什么文件。理解权限模型是避免"Permission denied"和防止安全漏洞的基础。

## 文件权限模型

### 基本权限（ugo+rwx）

```bash
ls -l /etc/passwd
-rw-r--r-- 1 root root 2.6K Jun 24 10:00 /etc/passwd
| 第一部分 | 第二部分 | 第三部分 |
| 权限位     | 属主    | 属组     |
```

权限位拆解（11 个字符）：

```
- r w -  r - -  r - -
↑ ↑  ↑  ↑  ↑  ↑  ↑
类型      属主(u)   属组(g)   其他(o)
d=目录       rwx      r-x      r-x
l=链接       读写执行  读执行  读执行
```

### chmod — 修改权限

```bash
# 符号模式（推荐，语义清晰）
chmod u+x script.sh             # 属主加执行权限
chmod g-w config.txt            # 属组去掉写权限
chmod o+r public.log            # 其他人加读权限
chmod u+x,g-w script.sh         # 组合写
chmod a+x script.sh             # 所有人（ugo）+ 执行

# 数字模式
# r=4 w=2 x=1
# rwx = 7, r-x = 5, r-- = 4
chmod 755 script.sh             # 属主全部，属组和他人读+执行
chmod 644 config.txt            # 属主读写，属组和他人只读
chmod 600 id_rsa                # 私钥只能属主读写
```

数字模式速查：

| 权限 | 数字 | 说明 |
|------|------|------|
| `rwx` | 7 | 全部权限 |
| `rw-` | 6 | 读写 |
| `r-x` | 5 | 读+执行 |
| `r--` | 4 | 只读 |
| `---` | 0 | 无权限 |

### chown — 修改属主

```bash
chown user1 file.txt                     # 改属主
chown user1:group1 file.txt              # 同时改属主和属组
chown -R user1:group1 /data/app/         # 递归修改
```

### umask — 默认权限

```bash
umask                    # 查看当前值（一般是 0022）
# 文件默认 666 - umask = 644
# 目录默认 777 - umask = 755
```

## 用户管理

### 用户操作

```bash
# 创建用户
useradd -m -s /bin/bash deploy          # -m 创建家目录，-s 指定 shell
passwd deploy                           # 设置密码

# 删除用户
userdel -r deploy                       # -r 一并删除家目录和邮件池

# 查看用户
whoami                                  # 当前用户名
id                                      # uid, gid, 所属组
cat /etc/passwd                         # 所有用户列表
```

`/etc/passwd` 每行格式：

```
username:x:uid:gid:description:/home/username:/bin/bash
```

### 组操作

```bash
groupadd devs                           # 创建组
usermod -aG devs user1                  # 把 user1 加到 devs 组
groups user1                            # 查看用户所属组
gpasswd -d user1 devs                   # 从组移除
```

### sudo — 提权执行

```bash
sudo systemctl restart nginx            # 以 root 执行单条命令
sudo -u deploy git pull                 # 以 deploy 用户执行
sudo -s                                 # 进入 root shell
sudo -i                                 # 进入 root 环境（类似 su）
```

sudo 配置在 `/etc/sudoers`，建议用 `visudo` 编辑：

```bash
visudo                                  # 安全编辑器（语法错误会报错）
# 常用配置：
# deploy    ALL=(ALL) ALL               # deploy 可以执行任何命令
# %devs     ALL=(ALL) NOPASSWD: ALL     # devs 组无需密码
# deploy    ALL=(ALL) /bin/systemctl    # 只允许执行 systemctl
```

## 特殊权限

### SUID / SGID / Sticky Bit

```bash
# SUID（s 出现在属主执行位）：以文件属主身份执行，不以执行者身份
chmod u+s /usr/bin/passwd
# 例子：普通用户改密码，passwd 以 root 身份改 /etc/shadow

# Sticky Bit（t 出现在其他执行位）：只能删自己的文件
chmod +t /tmp
# /tmp 目录所有人都能写，但只能删自己的文件
```

### 特殊文件：/etc/shadow

```bash
ls -l /etc/shadow
# ---------- 1 root root ...     # 普通用户完全不可见

# 查看密码信息（需要 root）
sudo cat /etc/shadow
# username:$6$salt$hash:lastchange:min:max:warn:inactive:expire
```

## 生产实践

### 最小权限原则

- 普通用户无法读写 `/etc/` 下的系统配置
- 配置文件设 `600` 或 `640`，不给 other 任何权限
- 私钥（`id_rsa`）必须是 `600`，否则 SSH 拒绝使用
- 进程以专用用户运行，不要全都用 root（nginx 用 www-data，MySQL 用 mysql）

### 常用操作

```bash
# 新建部署用户
useradd -m -s /bin/bash deploy
mkdir -p /data/app && chown deploy:deploy /data/app
# 加到 docker 组（不用 sudo 执行 docker）
usermod -aG docker deploy

# 给 deploy 免密 sudo
echo "deploy ALL=(ALL) NOPASSWD: ALL" > /etc/sudoers.d/deploy
```

### 踩坑记录

1. **私钥权限太宽松**：SSH 报 `Permissions 0644 for 'id_rsa' are too open`，直接拒绝连接。必须 `chmod 600 id_rsa`
2. **权限 777 滥用**：部署遇到权限问题，直接 `chmod -R 777`。这是安全灾难——任何用户都能改你的文件。正确做法：明确谁需要什么权限，用属主和属组控制
3. **`sudo -i` 切到 root 后忘了退出**：习惯性在 root 下执行普通操作，误删了系统文件。解决方案：养成 `sudo -i` 用完就 `exit` 的习惯，或只用 `sudo command` 提权单条命令
4. **用户删不掉**：`userdel deploy` 报错说有进程在运行。先 `pkill -u deploy` 杀掉后，再加 `-r` 删用户

## 总结

权限的核心是 **最小够用**——只给需要的，不给多余的。权限不够用 `chown` 改属主，不要 `chmod 777` 糊弄过去。
