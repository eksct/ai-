# Linux 文本查看与处理

## 概述

Linux 下没有"用鼠标打开文件"这种操作，所有文本处理都在终端里完成。查看日志、分析数据、过滤信息，靠的就是下面这些命令。

## 文本查看

### cat — 输出全部内容

```bash
cat nginx.conf                  # 看小文件
cat -n main.go                  # 显示行号
```

注意：大文件（几百 MB）别用 `cat`，终端会卡死。

### less — 分页查看（推荐）

```bash
less /var/log/syslog            # 分页查看大文件

# 快捷键
# 空格 / f     下一页
# b             上一页
# /keyword     向下搜索
# ?keyword     向上搜索
# n / N        下一个/上一个匹配
# g / G        跳转首行/尾行
# q            退出
```

`less` 比 `cat` 和 `more` 都强大——大文件瞬间打开，不读入全部内存。

### head / tail — 头尾查看

```bash
head -n 20 access.log           # 前 20 行
tail -n 50 access.log           # 后 50 行
tail -f access.log              # 实时跟踪（看日志滚动，Ctrl+C 退出）
```

**生产最常用组合：`tail -f`** 线上查问题必用。

## 文本过滤

### grep — 搜索文本

```bash
# 基本搜索
grep "ERROR" app.log            # 找包含 ERROR 的行
grep -i "error" app.log         # 忽略大小写
grep -v "DEBUG" app.log         # 排除 DEBUG 行
grep -c "ERROR" app.log         # 计数（多少行）
grep -n "ERROR" app.log         # 显示行号

# 递归搜索
grep -r "func main" ./src/      # 搜目录下所有文件
grep -rl "func main" ./src/     # 只显示文件名，不显示内容

# 正则搜索
grep "^2026-06-" app.log        # 2026年6月的日志
grep "5[0-9]{2}" app.log        # 500-599 的状态码

# 多条件
grep "ERROR" app.log | grep "timeout"   # 同时满足 ERROR + timeout
```

### sort — 排序

```bash
sort data.txt                            # 字典序排序
sort -n data.txt                         # 数字排序
sort -r data.txt                         # 逆序
sort -t: -k3 -n /etc/passwd              # 以 : 分隔，按第 3 列数字排序
```

### uniq — 去重

```bash
uniq data.txt                    # 去重（只能去相邻重复行）
sort data.txt | uniq             # 先排序再去重——这才是正确姿势
sort data.txt | uniq -c          # 去重并计数
sort data.txt | uniq -d          # 只显示重复的行
```

### wc — 统计

```bash
wc -l file.txt                   # 行数
wc -w file.txt                   # 单词数
wc -c file.txt                   # 字节数
wc -l */*.go                     # 统计 Go 文件总行数
```

### diff — 比较文件差异

```bash
diff a.txt b.txt                 # 上下文差异
diff -u a.txt b.txt              # unified 格式（更易读，git diff 风格）
diff -r dir1/ dir2/              # 递归比较目录
```

## 生产实践

### 常用组合

```bash
# 查错误日志
tail -f app.log | grep "ERROR"

# 统计访问量前 10 的 IP
cat access.log | awk '{print $1}' | sort | uniq -c | sort -nr | head -10

# 查进程是否存在
ps aux | grep nginx | grep -v grep

# 统计代码行数
find . -name "*.go" | xargs wc -l

# 找最近修改的文件
find . -name "*.conf" -mtime -1
```

### 踩坑记录

1. **在大文件上 `cat`**：`cat` 会把全部内容刷到终端，几百 MB 的文件让终端卡死。大文件用 `less`
2. **`grep` 匹配二进制文件**：`grep -r` 搜索包含 `.git` 和二进制文件的目录会很慢。加 `--exclude-dir=.git --binary-files=without-match`
3. **`tail -f` 把磁盘写满**：`tail -f > output.txt` 在后台跑了一周，output.txt 撑爆磁盘。用 `tail -f | grep --line-buffered` 做缓存限制

## 总结

`less` `grep` `tail -f` `sort | uniq -c` 这四个组合覆盖了日常 80% 的文本处理需求。看日志用 `tail -f`，搜内容用 `grep`，统计用 `sort | uniq -c`。
