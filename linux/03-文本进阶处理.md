# Linux 文本进阶处理

## 概述

`awk` 和 `sed` 是 Linux 文本处理的两大神器。`awk` 擅长**按列处理**（类似 Excel 的列操作），`sed` 擅长**按行替换**（查找替换的批处理版本）。日常不一定要手写复杂的 awk/sed 脚本，但理解它们的能力边界能帮你判断"这个问题用 shell 能不能一行搞定"。

## awk — 按列处理

### 基本原理

```bash
awk '{print $1, $3}' file.txt
#      ↑模式或条件  ↑每行默认按空格/制表符拆成列，$1 第1列，$2 第2列...
```

### 常见用法

```bash
# 取特定列
awk '{print $1, $NF}' access.log         # 第1列和最后一列
awk -F: '{print $1, $6}' /etc/passwd     # 以 : 分隔

# 条件过滤
awk '$9 > 500 {print $1, $9}' access.log # 状态码 > 500 的请求
awk '/ERROR/ {print $1, $2}' app.log     # 包含 ERROR 的行

# 统计
awk '{sum += $NF} END {print sum}' file.txt     # 最后一列求和
awk '{count[$1]++} END {for(k in count) print k, count[k]}' access.log
# ↑ 按 IP 统计访问次数

# 格式化输出
awk '{printf "%-20s %s\n", $1, $NF}' access.log
```

### 变量说明

| 变量 | 含义 |
|------|------|
| `$0` | 整行 |
| `$1`, `$2`... | 第 1、2... 列 |
| `NF` | 当前行的列数 |
| `NR` | 当前行号 |
| `FS` | 列分隔符（默认空白） |
| `OFS` | 输出列分隔符 |

## sed — 按行处理（替换、插入、删除）

### 基本原理

```bash
sed 's/旧/新/标志' file.txt
#    ↑操作  ↑查找替换的核心结构
```

### 常见用法

```bash
# 替换
sed 's/foo/bar/' config.txt             # 替换每行第一个 foo → bar
sed 's/foo/bar/g' config.txt            # 替换全部（global）
sed 's/foo/bar/2' config.txt            # 只替换每行第 2 个
sed 's/foo/bar/gi' config.txt           # 忽略大小写

# 行范围
sed '3s/foo/bar/' config.txt            # 只替换第 3 行
sed '1,5s/foo/bar/' config.txt          # 1~5 行
sed '/^#/s/foo/bar/' config.txt         # 只在注释行（# 开头）替换

# 删除
sed '/^$/d' file.txt                    # 删空行
sed '5,10d' file.txt                    # 删第 5~10 行
sed '/DEBUG/d' file.txt                 # 删包含 DEBUG 的行

# 插入
sed '3i\# new line' config.txt          # 在第 3 行前插入
sed '3a\# new line' config.txt          # 在第 3 行后追加
```

### 原地修改（-i）

```bash
sed -i 's/old/new/g' config.txt         # 直接改文件（危险）
sed -i.bak 's/old/new/g' config.txt     # 改之前备份为 config.txt.bak
# 生产环境永远加 .bak 后缀，防止改错了回不来
```

## cut — 简单列提取

```bash
cut -d: -f1,3 /etc/passwd              # 以 : 分隔，取第 1 和第 3 列
cut -c1-5 names.txt                     # 取前 5 个字符
```

`cut` 比 `awk` 简单轻量，适合纯按分隔符取列的简单场景。

## tr — 字符替换

```bash
echo "hello" | tr 'a-z' 'A-Z'          # 转大写 → HELLO
cat file.txt | tr -s '\n'              # 合并连续空行为一个
cat file.txt | tr -d '\r'              # 删 Windows 换行符（\r\n → \n）
```

```bash

# :set nu  显示行号
```

## 生产实践

### 一行命令解决问题

```bash
# 统计日志中各状态码数量
awk '{count[$9]++} END {for(k in count) print k, count[k]}' access.log

# 替换所有配置文件中的 IP
sed -i.bak 's/192\.168\.1\.1/192.168.2.1/g' /etc/nginx/conf.d/*.conf

# 取某个时间段的日志
sed -n '/2026-06-24 10:00/,/2026-06-24 11:00/p' app.log

# 提取所有 URL 路径
grep -oP '"GET \K[^? ]+' access.log | sort | uniq -c | sort -nr
```

### 管道思维

把复杂问题拆成一个个小步骤，用 `|` 串起来：

```
cat access.log | grep "ERROR" | awk '{print $1}' | sort | uniq -c | sort -nr | head -10
#  读日志    筛错误行     取IP列      排序   去重计数   按次数降序  前10名
```

### 踩坑记录

1. **`sed -i` 改错文件没有备份**：配置写错了，服务起不来。解决方案：永远加 `.bak` 后缀，或先用 `sed 's/.../.../'`（不加 -i）预览结果
2. **awk 分隔符不匹配**：日志字段用空格分隔，但是字段里也有空格（如时间字段）。解决方案：先确认分隔符格式，`awk '{print $NF}'` 如果不对就用 `-F'[ \t]+'`
3. **vim 不会退出**：新手按 Ctrl+Z 误操作挂起 vim，反复开多个 vim 进程。解决方案：学 `:q` 退出，真挂起了用 `fg` 恢复或 `kill %1` 杀掉

## 总结

`awk` 做列统计、`sed` 做行替换、管道串起多个命令解决复杂问题。能三行 shell 搞定的，不要写 50 行 Python 脚本。
