# Linux 文件与目录操作

## 概述

文件系统是 Linux 一切的基础——"一切皆文件"，目录、设备、网络 socket 都以文件形式暴露。掌握文件与目录操作是使用 Linux 的第一步。

## 目录结构速览

```
/          根目录
├── root   root 用户的家目录
├── home   普通用户的家目录 (/home/ubuntu, /home/ec2-user)
├── etc    系统配置文件 (/etc/nginx/nginx.conf)
├── var    可变数据 (/var/log/syslog, /var/lib/mysql)
├── tmp    临时文件（重启清空）
├── usr    用户程序 (/usr/bin, /usr/local)
├── opt    第三方软件
└── proc   进程虚拟文件系统（内存映射）
```

**只需要记住 `etc`（配置）、`var`（日志/数据）、`home`（用户目录）**，其他遇到再查。

## 核心命令

### pwd — 当前在哪

```bash
pwd
# /home/user/projects
```

### ls — 列出目录内容

```bash
ls                          # 列出文件名
ls -l                       # 详细信息（权限、大小、时间）
ls -a                       # 包含隐藏文件（. 开头的）
ls -lh                      # 人类可读大小（KB/MB/GB）
ls -lt                      # 按修改时间排序
ls -ltr                     # 按时间反序（最新的在最后）
```

`ls -l` 输出解读：

```
-rw-r--r-- 1 root root 4.0K Jun 24 10:00 filename
| 权限    | 链接数 | 属主 | 属组 | 大小 | 修改时间     | 文件名
```

### cd — 切换目录

```bash
cd /etc/nginx               # 绝对路径
cd ../                      # 上级目录
cd ~                        # 回自己的家目录
cd -                        # 回到上一个目录（来回切很实用）
```

### mkdir — 创建目录

```bash
mkdir logs                  # 创建单级
mkdir -p a/b/c/d            # 递归创建多级
mkdir -p app/{src,test,docs} # 批量创建 app/src app/test app/docs
```

### touch — 创建文件或更新时间戳

```bash
touch main.go               # 文件不存在则创建，存在则更新时间
```

### cp — 复制

```bash
cp a.txt b.txt              # 复制文件
cp -r src/ dst/             # 递归复制目录
cp -a src/ dst/             # 归档复制（保留权限和时间）
```

### mv — 移动或重命名

```bash
mv a.txt b.txt              # 重命名（同目录下）
mv a.txt /tmp/              # 移动文件到 /tmp
mv dir1/ dir2/              # 移动/重命名目录
```

### rm — 删除

```bash
rm a.txt                    # 删文件
rm -r dir/                  # 递归删目录
rm -rf dir/                 # 强制删，不确认（"rm -rf /" 是自杀命令）
```

> `rm -rf` 是 Linux 最危险的命令之一，没有回收站，删了就是真的没了。每次敲 `rm -rf` 之前确认三秒。

### find — 查找文件

```bash
# 按名查找
find /etc -name "nginx.conf"

# 按类型查找
find / -type d -name "logs"         # 目录
find / -type f -name "*.log"        # 文件

# 按时间查找
find /var/log -mtime -7             # 7 天内改过的文件
find /var/log -mtime +30            # 30 天前的文件

# 查找并执行
find /tmp -name "*.tmp" -delete     # 找到直接删
find /var/log -name "*.log" -exec grep "error" {} \;
```

### tree — 目录树

```bash
tree                            # 显示目录树
tree -L 2                       # 只展开 2 层
```

有些系统没装 `tree`，可以用 `find . -type f | head -20` 替代。

## 文件类型

```bash
ls -l                            # 第一个字符标识类型
# -rw-r--r--   普通文件
# drwxr-xr-x   目录
# lrwxrwxrwx   符号链接
# crw-r--r--   字符设备
# brw-r--r--   块设备
```

### 软链接 vs 硬链接

```bash
# 软链接（符号链接）——类似 Windows 快捷方式
ln -s /data/storage ./link_to_storage
# 删除源文件，链接断掉

# 硬链接——同一个文件的多个名字
ln /data/file ./hard_link
# 删一个，另一个还在（引用计数减 1）
```

软链接常用，硬链接基本用不上——但知道它能帮你理解 Linux 文件系统的引用计数机制。

## 生产实践

### 常用组合

- **快速查看目录结构**：`ls -la` / `tree -L 2`
- **查找大文件**：`find / -type f -size +100M -exec ls -lh {} \;`
- **安全删除**：`mv /path/to/delete /tmp/`（先移再删，后悔了还能从 /tmp 找回来）
- **批量创建目录**：`mkdir -p app/{src,test,docs,config}`

### 踩坑记录

1. **`rm -rf` 删错了**：线上执行 `rm -rf /var/log/app /` （多了一个空格，把根目录也删了）。解决方案：`rm` 时用绝对路径 + 再三确认，重要目录配 `chattr +i` 保护
2. **磁盘写满是因为没清理日志**：`/var/log` 下的日志只增不减，docker 容器日志更是磁盘杀手。解决方案：配置 logrotate 自动轮转，限制单个日志大小
3. **软链接的坑**：`cp -r` 复制目录时软链接默认被解引用（复制目标文件而非链接本身）。用 `cp -a` 保留软链接。同理 `tar` 打包时注意 `--dereference`

## 总结

`ls` `cd` `cp` `mv` `rm` `find` 是最高频的 6 个命令，每天会用几十次。`rm -rf` 是唯一需要害怕的。记住 `-r` 递归和 `-f` 强制的组合可以毁掉整个系统。
