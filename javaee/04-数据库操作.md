# 数据库操作（MyBatis-Plus）

## 概述

国内生产环境数据持久化选型，**MyBatis-Plus 是事实标准**。JPA 在国外流行，但国内因为复杂 SQL 多、DBA 习惯看原生 SQL，MyBatis 系更吃香。MyBatis-Plus 在 MyBatis 基础上做了大量增强，代码生成器 + 自动分页 + Lambda 查询，让单表操作基本不用写 SQL。

## 为什么不选 JPA？

| 维度 | MyBatis-Plus | JPA/Hibernate |
|------|-------------|---------------|
| SQL 控制 | 完全掌控 | 自动生成，难优化 |
| 复杂查询 | xml 写原生 SQL | JPQL/Criteria 不好写 |
| 性能 | N+1 需自己注意 | 自动关联容易 N+1 |
| 学习曲线 | 低 | 高（缓存/懒加载/状态管理） |
| 国内生态 | 主流 | 少数新项目在用 |

**一句话：** 需要复杂 SQL 优化 → MP，纯 CRUD 无复杂 SQL → JPA 也行。

## 实体与 Mapper

### 实体类

```java
@Data
@TableName("`order`")   // 表名是关键字时加反引号
public class Order {
    @TableId(type = IdType.ASSIGN_ID)
    private Long id;

    private String orderNo;
    private Long userId;
    private BigDecimal totalAmount;
    private Integer status;

    @TableField(fill = FieldFill.INSERT)
    private LocalDateTime createTime;

    @TableField(fill = FieldFill.INSERT_UPDATE)
    private LocalDateTime updateTime;

    @TableLogic
    private Integer deleted;
}
```

## 常用操作

### 增删改查

```java
@Service
public class OrderService {
    @Autowired
    private OrderMapper orderMapper;

    // 新增
    public void create(Order order) {
        orderMapper.insert(order);  // ID 会自动回填
    }

    // 查询
    public Order getById(Long id) {
        return orderMapper.selectById(id);
    }

    public List<Order> listByUserId(Long userId) {
        return orderMapper.selectList(
            Wrappers.<Order>lambdaQuery()
                .eq(Order::getUserId, userId)
                .orderByDesc(Order::getCreateTime)
        );
    }

    // 更新
    public void updateStatus(Long id, Integer status) {
        orderMapper.update(
            Wrappers.<Order>lambdaUpdate()
                .set(Order::getStatus, status)
                .eq(Order::getId, id)
        );
    }

    // 删除（逻辑删除）
    public void delete(Long id) {
        orderMapper.deleteById(id);  // 实际执行 UPDATE SET deleted=1
    }
}
```

### 分页

```java
@Configuration
public class MybatisPlusConfig {
    @Bean
    public MybatisPlusInterceptor mybatisPlusInterceptor() {
        MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
        interceptor.addInnerInterceptor(new PaginationInnerInterceptor(DbType.MYSQL));
        return interceptor;
    }
}

// 使用
public PageResult<OrderVO> listOrders(OrderQueryReq req) {
    Page<Order> page = new Page<>(req.getPage(), req.getSize());
    // 条件构造
    LambdaQueryWrapper<Order> wrapper = Wrappers.<Order>lambdaQuery()
        .eq(req.getUserId() != null, Order::getUserId, req.getUserId())
        .eq(req.getStatus() != null, Order::getStatus, req.getStatus())
        .ge(req.getStartTime() != null, Order::getCreateTime, req.getStartTime())
        .le(req.getEndTime() != null, Order::getCreateTime, req.getEndTime())
        .orderByDesc(Order::getCreateTime);

    IPage<Order> result = orderMapper.selectPage(page, wrapper);
    return PageResult.of(result);
}
```

## XML 复杂查询

```xml
<!-- OrderMapper.xml -->
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<mapper namespace="com.example.order.dal.mapper.OrderMapper">

    <!-- 分页 + 联表查询 -->
    <select id="selectPageWithUser" resultType="com.example.order.biz.model.vo.OrderVO">
        SELECT o.*, u.username, u.phone
        FROM `order` o
        LEFT JOIN `user` u ON o.user_id = u.id
        <where>
            <if test="query.status != null">
                AND o.status = #{query.status}
            </if>
            <if test="query.userId != null">
                AND o.user_id = #{query.userId}
            </if>
            <if test="query.startTime != null">
                AND o.create_time >= #{query.startTime}
            </if>
            <if test="query.endTime != null">
                AND o.create_time &lt;= #{query.endTime}
            </if>
        </where>
        ORDER BY o.create_time DESC
    </select>

    <!-- 批量更新 -->
    <update id="batchUpdateStatus">
        UPDATE `order`
        SET status = #{status}, update_time = NOW()
        WHERE id IN
        <foreach collection="ids" item="id" open="(" separator="," close=")">
            #{id}
        </foreach>
    </update>
</mapper>
```

## 踩坑记录

### 1. 大分页性能问题

```sql
-- ❌ 传统分页越往后越慢
SELECT * FROM `order` LIMIT 100000, 20;
```

**原因：** MySQL 需要扫描 100020 行再丢掉前 100000 行

**优化方案：**
```sql
-- ✅ 子查询优化
SELECT * FROM `order`
WHERE id > (SELECT id FROM `order` ORDER BY id LIMIT 100000, 1)
ORDER BY id LIMIT 20;

-- ✅ 或者带上排序字段的索引，用 between 分页
SELECT * FROM `order` WHERE id BETWEEN 100001 AND 100020 ORDER BY id;
```

MyBatis-Plus 也支持：
```java
// 用 last 方法拼接自定义 SQL
page.optimizeCountSql(true);           // 优化 count 查询
// 或使用 SafePaginator
```

### 2. MyBatis-Plus 逻辑删除 + 唯一索引冲突

**现象：** 同一个手机号注册两次，第一次逻辑删除后第二次注册报唯一索引冲突

**原因：** 逻辑删除只是把 `deleted` 置为 1，但唯一索引还是存在

**解决：**
- 方案一：联合唯一索引包含 `deleted` 字段（MySQL 允许 NULL 不参与唯一约束）
- 方案二：插入前手动检查
- 方案三：物理删除（如果业务允许）

### 3. N+1 查询

**现象：** 查 100 个订单，每个订单又查一次用户 → 101 条 SQL

**解决：** 联表查询或用 MP 的 `@TableName(autoResultMap = true)` + 嵌套查询

### 4. $ 符号注入

```xml
<!-- ❌ 极度危险：SQL 注入 -->
<if test="orderBy != null">
    ORDER BY ${orderBy}
</if>

<!-- ✅ 用 #{} 安全参数化 -->
<if test="orderBy != null">
    ORDER BY ${orderBy}  <!-- 这个确实需要 ${}，但必须校验取值 -->
</if>
```

**生产做法：** 前端传排序字段时，**白名单校验**
```java
List<String> allowedFields = Arrays.asList("create_time", "amount", "status");
if (!allowedFields.contains(orderBy)) {
    throw new BusinessException(400, "不支持的排序字段");
}
```
