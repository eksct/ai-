# 安全认证

## 概述

Java 生产中最常用的认证方案：**Spring Security + JWT + RBAC**。不要自己去实现加密和认证逻辑——Spring Security 虽然复杂，但它把那些安全边界都考虑到了。

## 认证 vs 授权

```
认证（Authentication）：你是谁？    → 登录 → 生成 JWT
授权（Authorization）：你能做什么？ → JWT 解析 → RBAC 校验
```

### Token 生成和校验

```java
@Component
public class JwtUtils {
    @Value("${jwt.secret}")
    private String secret;

    @Value("${jwt.expire:86400}")
    private long expire;

    private SecretKey getSecretKey() {
        return Keys.hmacShaKeyFor(secret.getBytes(StandardCharsets.UTF_8)); // HS256 要求密钥至少 32 字节
    }

    /**
     * 生成 Token
     * @param userId 用户 ID
     * @param roles  角色列表
     */
    public String generateToken(Long userId, List<String> roles) {
        return Jwts.builder()
                .subject(userId.toString())
                .claim("roles", roles)
                .issuedAt(new Date())
                .expiration(new Date(System.currentTimeMillis() + expire * 1000))
                .signWith(getSecretKey())
                .compact();
    }

    /**
     * 生成 Token（自定义过期时间）
     */
    public String generateToken(Long userId, List<String> roles, long expireSeconds) {
        return Jwts.builder()
                .subject(userId.toString())
                .claim("roles", roles)
                .issuedAt(new Date())
                .expiration(new Date(System.currentTimeMillis() + expireSeconds * 1000))
                .signWith(getSecretKey())
                .compact();
    }

    /**
     * 解析 Token
     */
    public Claims parseToken(String token) {
        try {
            return Jwts.parser()
                    .verifyWith(getSecretKey())
                    .build()
                    .parseSignedClaims(token)
                    .getPayload();
        } catch (JwtException e) {
            throw new BusinessException(401, "Token 无效或已过期");
        }
    }

    /**
     * 判断 Token 是否即将过期（用于自动续期）
     */
    public boolean isAboutToExpire(String token) {
        Claims claims = parseToken(token);
        long remaining = claims.getExpiration().getTime() - System.currentTimeMillis();
        return remaining < expire * 1000 / 2; // 过期时间少于一半时续期
    }
}
```

## Spring Security 集成

### 核心配置

```java
@Configuration
@EnableWebSecurity
@EnableMethodSecurity
public class SecurityConfig {
    @Autowired
    private JwtAuthFilter jwtAuthFilter;

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
            .csrf(csrf -> csrf.disable())
            .sessionManagement(session ->
                session.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
            .authorizeHttpRequests(auth -> auth
                // 白名单
                .requestMatchers("/api/auth/login").permitAll()
                .requestMatchers("/api/auth/register").permitAll()
                .requestMatchers("/api/auth/refresh").permitAll()
                .requestMatchers("/doc.html", "/v3/api-docs/**").permitAll()
                .requestMatchers("/actuator/health").permitAll()
                // 其他接口需要认证
                .anyRequest().authenticated()
            )
            .addFilterBefore(jwtAuthFilter, UsernamePasswordAuthenticationFilter.class)
            .exceptionHandling(ex -> ex
                .authenticationEntryPoint((request, response, authException) -> {
                    response.setContentType("application/json;charset=utf-8");
                    response.getWriter().write(JSON.toJSONString(
                        ApiResult.error(401, "未登录或 Token 已过期")));
                })
                .accessDeniedHandler((request, response, accessDeniedException) -> {
                    response.setContentType("application/json;charset=utf-8");
                    response.getWriter().write(JSON.toJSONString(
                        ApiResult.error(403, "无权限")));
                })
            );
        return http.build();
    }

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();  // 不要用 MD5！
    }
}
```

### JWT 认证过滤器

```java
@Component
public class JwtAuthFilter extends OncePerRequestFilter {
    @Autowired
    private JwtUtils jwtUtils;

    private static final List<String> WHITE_LIST = Arrays.asList(
        "/api/auth/login", "/api/auth/register",
        "/doc.html", "/v3/api-docs", "/actuator/health"
    );

    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                     HttpServletResponse response,
                                     FilterChain chain) throws ServletException, IOException {
        String path = request.getRequestURI();
        // 白名单跳过
        if (WHITE_LIST.stream().anyMatch(path::startsWith)) {
            chain.doFilter(request, response);
            return;
        }

        String token = extractToken(request);
        if (token == null) {
            chain.doFilter(request, response);
            return;
        }

        Claims claims = jwtUtils.parseToken(token);
        List<String> roles = claims.get("roles", List.class);

        // 构建 Authentication 放入上下文
        UsernamePasswordAuthenticationToken auth =
            new UsernamePasswordAuthenticationToken(
                claims.getSubject(), null,
                roles.stream().map(SimpleGrantedAuthority::new).toList()
            );
        auth.setDetails(new WebAuthenticationDetailsSource().buildDetails(request));
        SecurityContextHolder.getContext().setAuthentication(auth);

        chain.doFilter(request, response);
    }

    private String extractToken(HttpServletRequest request) {
        String header = request.getHeader("Authorization");
        if (header != null && header.startsWith("Bearer ")) {
            return header.substring(7);
        }
        return null;
    }
}
```

## RBAC（权限控制）

### 数据库设计

```sql
-- 五张标准 RBAC 表
CREATE TABLE `user` (
    `id` bigint NOT NULL,
    `username` varchar(64) NOT NULL,
    `password` varchar(256) NOT NULL,
    PRIMARY KEY (`id`)
);

CREATE TABLE `role` (
    `id` bigint NOT NULL,
    `code` varchar(64) NOT NULL COMMENT '角色编码: ADMIN/OPERATOR/VIEWER',
    `name` varchar(64) NOT NULL,
    PRIMARY KEY (`id`)
);

CREATE TABLE `permission` (
    `id` bigint NOT NULL,
    `code` varchar(128) NOT NULL COMMENT '权限编码: order:create, user:delete',
    `name` varchar(64) NOT NULL,
    PRIMARY KEY (`id`)
);

CREATE TABLE `user_role` (
    `user_id` bigint NOT NULL,
    `role_id` bigint NOT NULL,
    PRIMARY KEY (`user_id`, `role_id`)
);

CREATE TABLE `role_permission` (
    `role_id` bigint NOT NULL,
    `permission_id` bigint NOT NULL,
    PRIMARY KEY (`role_id`, `permission_id`)
);
```

## 密码安全

```java
// 注册时加密
public void register(RegisterReq req) {
    if (userMapper.existsByUsername(req.getUsername())) {
        throw new BusinessException(400, "用户名已存在");
    }

    User user = new User();
    user.setUsername(req.getUsername());
    user.setPassword(passwordEncoder.encode(req.getPassword())); // BCrypt 加密
    userMapper.insert(user);
}

// 登录时校验
public String login(LoginReq req) {
    User user = userMapper.findByUsername(req.getUsername());
    if (user == null || !passwordEncoder.matches(req.getPassword(), user.getPassword())) {
        throw new BusinessException(401, "用户名或密码错误");
    }
    // 生成 Token
    return jwtUtils.generateToken(user.getId(), user.getRoles());
}
```

**为什么不用 MD5？** MD5 是哈希函数不是加密，彩虹表可以反查。BCrypt 自动加盐、可调强度、抗暴力破解。

## 踩坑记录

### 1. Token 过期后用户无感续期

**场景：** 用户正在填写一个长表单，提交时 Token 过期了，被踢回登录页

**方案：** 双 Token 机制

```java
// 登录时返回两个 token
public LoginVO login(LoginReq req) {
    // ...校验用户名密码
    String accessToken = jwtUtils.generateToken(user.getId(), user.getRoles(), 3600);      // 1h
    String refreshToken = jwtUtils.generateToken(user.getId(), user.getRoles(), 2592000); // 30d
    return new LoginVO(accessToken, refreshToken);
}

// 刷新接口
@PostMapping("/refresh")
public ApiResult<LoginVO> refresh(@RequestBody RefreshReq req) {
    Claims claims = jwtUtils.parseToken(req.getRefreshToken());
    String newAccessToken = jwtUtils.generateToken(
        Long.valueOf(claims.getSubject()),
        claims.get("roles", List.class),
        3600
    );
    return ApiResult.success(new LoginVO(newAccessToken, null));
}
```

### 2. 密码明文传输

**现象：** 通过抓包可以看到用户密码明文

**解决：** HTTPS 是基础。前端再加一层 RSA 加密，后端解密。

### 3. 权限数据放 Token 里导致 Token 太大

**现象：** 某个管理员有几万个权限，Token 几十 KB，每次请求都携带

**解决：** Token 里只放角色编码，权限列表在服务端缓存（Redis）

### 4. 退出登录后端要怎么处理

JWT 是无状态的，服务端无法主动让 Token 失效。做法：
- **短期有效期**（1-2 小时）：Token 自然过期
- **黑名单**：退出时把 Token 加入 Redis 黑名单
- **refreshToken + accessToken 双 Token**：refreshToken 被收回时 accessToken 无法刷新
